# 【インターフェイス仕様書】報告書プレビュー

## 1. 処理概要

- **出力の目的**：PDF出力前に報告書の内容をブラウザ上でプレビュー表示し、内容を確認できるようにする。
- **出力形式**：JSON（HTMLコンテンツを含む、または構造化データとしてフロントエンドでレンダリング）
- **対象データ範囲**：指定プロジェクトの指定期間（`start_date`〜`end_date`）に開始日または終了日が含まれるタスク。
- **実行タイミング**：`GET /api/v1/projects/{project_id}/reports/generate/?start_date=...&end_date=...` 呼び出し時（報告書作成画面のプレビューボタン押下）
- **ソート順**：`order`（並び順）昇順

## 3. 備考

- このAPIはファイルダウンロードではなくJSONレスポンスを返す。フロントエンドがこのデータを使用してプレビューをレンダリングする。
- タスクが0件の場合、`tasks` は空配列、集計値はすべて0を返す。
- `tasks` の並び順は `order` 昇順。階層構造はフロントエンド側でツリー表示する。
- プレビューのレンダリング結果とPDF出力結果は同一のデータ源を使用するため、表示内容は一致する。
