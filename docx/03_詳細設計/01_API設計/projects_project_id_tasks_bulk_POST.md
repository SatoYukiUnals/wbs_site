# 【API設計書】タスク一括作成

## 1. 概要

指定プロジェクトに複数タスクを一括で作成する。主にWBSテンプレート適用以外の用途で使用。admin 以上のみ実行可能。全タスクの階層チェックを事前に行い、いずれかが失敗した場合は全件ロールバックする。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/tasks/bulk/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `tasks`（タスクリスト）

- **型:** `array` / **必須:** ○
- 1件以上

各要素：

| フィールド | 型 | 必須 | 備考 |
| :--- | :--- | :--- | :--- |
| `title` | string | ○ | 1〜200文字 |
| `parent_task_id` | integer | - | 同プロジェクト内のタスクID |
| `start_date` | date | - | YYYY-MM-DD |
| `end_date` | date | - | `start_date` 以降 |
| `estimated_hours` | number | - | 0以上 |
| `priority` | string | - | 高/中/低 |
| `quarter_id` | integer | - | |

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "tasks": [
    {
      "id": 11,
      "title": "フロントエンド実装",
      "parent_task_id": 5,
      "depth": 3,
      "status": "未着手",
      "progress": 0
    },
    {
      "id": 12,
      "title": "バックエンド実装",
      "parent_task_id": 5,
      "depth": 3,
      "status": "未着手",
      "progress": 0
    }
  ]
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 階層超過 | 400 | `ERR_VAL_008` | タスクの階層は5層までです | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション（全件事前チェック）**
   - 各タスクの `title` 存在・長さチェック
   - 各タスクの `parent_task_id` が同プロジェクト内に存在することを確認
   - 各タスクの深さが5層以内であることを確認
     - いずれかでエラー：400 `ERR_VAL_001` / `ERR_VAL_008` を返す
3. **DB操作（トランザクション）**
   - 全タスクを一括 INSERT
   - いずれかでエラー時はロールバック
     - DBエラー：500 `ERR_SYS_500` を返す
4. **レスポンス生成**
   - 作成したタスク一覧を返却
