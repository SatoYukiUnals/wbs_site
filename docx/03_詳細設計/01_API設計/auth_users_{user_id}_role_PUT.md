# 【API設計書】ユーザーロール変更API

## 1. 概要

テナント内ユーザーのロールを変更する。master のみ実行可能。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/auth/users/{user_id}/role/`
- **認証:** 必要（Bearer トークン）
- **権限:** master のみ

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `user_id` | UUID | ✅ | 変更対象ユーザーID |

### ボディ（JSON）

#### `role`

- **型:** `string` / **必須:** ✅
- `admin` または `member`（`master` への変更は不可）

```json
{
  "role": "admin"
}
```

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "user_id": "uuid",
  "role": "admin"
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 対象ユーザーが存在しない・テナント外 | 404 | 対象ユーザーが見つかりません |
| `master` への変更 | 400 | masterロールへの変更はできません |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - `role=master` のみ実行可。それ以外：403 を返す
2. **ユーザー検索**
   - `user_id` かつ `tenant_id` で検索。見つからない：404 を返す
3. **バリデーション**
   - `role` が `master` の場合：400 を返す
4. **DB更新**
   - `user.role` を UPDATE
5. **レスポンス**
   - 更新後情報を返す
