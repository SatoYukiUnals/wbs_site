# 【API設計書】プロジェクトメンバー追加

## 1. 概要

同テナント内の既存ユーザーを指定プロジェクトのメンバーとして追加する。プロジェクト内の admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/members/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `user_id`（追加するユーザーID）

- **型:** `integer` / **必須:** ○
- 同テナント内のユーザーであること

#### `role`（プロジェクト内ロール）

- **型:** `string` / **必須:** ○
- `admin` または `member` のいずれか

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "user_id": 3,
  "username": "jiro",
  "email": "jiro@example.com",
  "role": "member"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 対象ユーザー未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| 既にメンバー | 400 | `ERR_VAL_006` | このユーザーは既にメンバーです | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - `user_id`・`role` の存在・値チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **ユーザー存在確認**
   - `User` テーブルで `user_id` AND `tenant_id` で SELECT
     - 未存在 or 別テナント：404 `ERR_DAT_404` を返す
4. **重複確認**
   - `ProjectMember` で `project_id` AND `user_id` が既存でないことを確認
     - 重複：400 `ERR_VAL_006` を返す
5. **DB操作**
   - `ProjectMember` レコードを INSERT
     - DBエラー：500 `ERR_SYS_500` を返す
6. **レスポンス生成**
   - 追加したメンバー情報を返却
