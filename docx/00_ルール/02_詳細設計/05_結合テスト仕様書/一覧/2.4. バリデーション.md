# 2.4. バリデーション

> キーワード検索欄に対するバリデーションのテストケース。
> 実際の仕様書作成時は、対象画面の文字数制限・単語数上限を確認して値を置き換えること。

---

### ■ 2.4.1. 空欄
#### ▶ 前提条件
- 画面名：
- ログインユーザー：
- 操作対象データ：

#### ▶ テスト内容
1. 全検索欄を空欄のまま「検索」ボタンを押下する

#### ▶ 期待結果
- エラーメッセージが表示されないこと
- 全件が表示されること

---

### ■ 2.4.2. 記号・絵文字
#### ▶ 前提条件
- 画面名：
- ログインユーザー：
- 操作対象データ：

#### ▶ テスト内容
1. 「キーワード検索」欄に以下の記号を入力して検索する
   ```
   !@#$%^&*()
   ```
2. 「キーワード検索」欄に以下の絵文字を入力して検索する
   ```
   😀🎉
   ```

#### ▶ 期待結果
- ページクラッシュやエラー画面が表示されないこと
- 検索が実行されること
- 一致データがあれば表示され、なければ0件と表示されること

---

### ■ 2.4.3. XSS
#### ▶ 前提条件
- 画面名：
- ログインユーザー：
- 操作対象データ：

#### ▶ テスト内容
1. 「キーワード検索」欄に以下を入力して検索する
   ```
   <script>alert(1)</script>
   ```
2. 「キーワード検索」欄に以下を入力して検索する
   ```
   "><img src=x onerror=alert(1)>
   ```

#### ▶ 期待結果
- JavaScriptが実行されないこと（ダイアログ等が表示されない）
- 入力値が `&lt;script&gt;` 等エスケープされた文字列として画面に表示されること
