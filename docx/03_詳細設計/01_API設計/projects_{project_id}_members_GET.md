# 【API設計書】メンバー一覧API

## 1. 概要

プロジェクトのメンバー一覧とロールを取得する。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/{project_id}/members/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクトメンバー以上

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `project_id` | UUID | ✅ | 対象プロジェクトID |

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "members": [
    {
      "user_id": "uuid",
      "username": "山田太郎",
      "email": "yamada@example.com",
      "role": "admin"
    }
  ]
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| プロジェクト存在しない | 404 | プロジェクトが見つかりません |
| メンバーでない | 403 | このプロジェクトへのアクセス権がありません |

## 5. 内部処理 / ロジック

1. **アクセス確認**
   - プロジェクト存在・テナント一致・メンバーであることを確認
2. **データ取得**
   - `project_member` と `user` を JOIN して一覧取得
3. **レスポンス**
   - メンバー一覧を返す
