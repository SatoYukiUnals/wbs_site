# 【API設計書】ユーザー削除API

## 1. 概要

テナント内ユーザーを削除する。master のみ実行可能。自分自身・他の master は削除不可。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/auth/users/{user_id}/`
- **認証:** 必要（Bearer トークン）
- **権限:** master のみ

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `user_id` | UUID | ✅ | 削除対象ユーザーID |

## 4. レスポンス

### 成功 (204 No Content)

ボディなし

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 対象ユーザーが存在しない・テナント外 | 404 | 対象ユーザーが見つかりません |
| 自分自身を削除しようとした | 400 | 自分自身を削除することはできません |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - `role=master` のみ実行可。それ以外：403 を返す
2. **ユーザー検索**
   - `user_id` かつ `tenant_id` で検索。見つからない：404 を返す
3. **バリデーション**
   - 自分自身（`user_id` == JWT の `user_id`）の場合：400 を返す
4. **DB削除**
   - `user` レコードを物理削除（または `deleted_at` 付与で論理削除）
5. **レスポンス**
   - 204 No Content を返す
