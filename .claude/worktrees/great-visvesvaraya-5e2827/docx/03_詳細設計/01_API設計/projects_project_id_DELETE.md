# 【API設計書】プロジェクト削除

## 1. 概要

指定プロジェクトを論理削除する。プロジェクト内の admin 以上のみ実行可能。論理削除のため `deleted_at` を設定し、一覧取得からは除外される。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/projects/{project_id}/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

## 4. レスポンス

### 成功 (204 No Content)

レスポンスボディなし

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `tenant_id`・`user_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **プロジェクト取得・権限確認**
   - `Project` を `project_id` AND `tenant_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
   - `ProjectMember` で `role` が `admin` 以上であることを確認
     - 権限不足：403 `ERR_PERM_001` を返す
3. **論理削除**
   - `Project.deleted_at` に現在日時を UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
4. **レスポンス生成**
   - 204 No Content を返却
