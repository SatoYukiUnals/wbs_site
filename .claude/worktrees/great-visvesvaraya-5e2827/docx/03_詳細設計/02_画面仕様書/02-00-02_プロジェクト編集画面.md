# 【画面仕様書】プロジェクト編集画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 02-00-02 |
| 画面名 | プロジェクト編集画面 |
| URL | `/projects/{id}/edit` |
| 対応デザイン | - |

## 2. 目的

既存プロジェクトの名称・説明・開始日・終了日を編集する。admin以上のみ操作可能。

## 3. 処理概要

### 画面で操作できる情報

- プロジェクト名・説明・開始日・終了日の編集
- 変更の保存

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| プロジェクト詳細画面 | 編集ボタン | プロジェクト編集画面 |
| プロジェクト編集画面 | 保存成功 | プロジェクト詳細画面 |
| プロジェクト編集画面 | キャンセルボタン | プロジェクト詳細画面 |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| プロジェクト名 | submit | 必須・1〜200文字 | ERR_VAL_001 | プロジェクト名を入力してください |
| 開始日 | submit | 必須・日付形式 | ERR_VAL_001 | 開始日を入力してください |
| 終了日 | submit | 開始日以降（入力時） | ERR_VAL_001 | 終了日は開始日以降の日付を入力してください |

## 8. アクション

### 画面表示時

1. `GET /api/v1/projects/{id}/` 呼び出し
2. 取得データをフォームに設定

### 保存ボタン押下時

1. フロント側バリデーション実行
2. `PUT /api/v1/projects/{id}/` 呼び出し
3. 成功（200）: `/projects/{id}` へ遷移し、成功トーストを表示
4. 失敗（400）: エラーメッセージを表示
