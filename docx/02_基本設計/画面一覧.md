# 画面一覧

WBS管理ソフトの全画面を一覧化する。詳細設計フェーズの画面仕様書は本ファイルで定義した画面ID・画面名を正として作成する。

- **作成日**：2026-04-23
- **バージョン**：1.0
- **画面総数**：32画面

---

## 1. 画面ID体系

画面IDは **3階層** の数字体系で表現する。

```
[カテゴリ番号]-[データ単位番号]-[操作区分]
   (2桁)         (2桁)           (2桁)
```

### 1.1 カテゴリ番号（第1階層）

機能大分類を表す。

| 番号 | カテゴリ | 対応機能 |
|------|---------|---------|
| `01` | 認証・ユーザー管理 | 3-1 |
| `02` | プロジェクト管理 | 3-2 |
| `03` | クォーター管理 | 3-3 |
| `04` | WBS・タスク管理 | 3-4, 3-5 |
| `05` | ガントチャート | 3-6 |
| `06` | プロダクトロードマップ | 3-7 |
| `07` | ダッシュボード | 3-8 |
| `08` | レビュー管理 | 3-9 |
| `09` | 報告書管理 | 3-10 |
| `10` | Excel出力 | 3-11 |
| `11` | テンプレート管理 | 3-12 |

### 1.2 データ単位番号（第2階層）

カテゴリ内の主要データ／サブ領域を区切る。カテゴリごとに採番を行う（詳細は §3 参照）。

### 1.3 操作区分（第3階層）

画面の操作種別を表す。

| 番号 | 操作 | 意味 |
|------|------|------|
| `00` | 一覧 | 複数件の検索・表示 |
| `01` | 登録 | 新規作成 |
| `02` | 変更 | 既存データの編集 |
| `03` | 詳細 | 1件の詳細表示 |
| `04` | その他 | 上記以外の特殊操作（ログイン、プレビュー、設定、出力設定 等） |
| `05` 以降 | 拡張 | 必要に応じて割当 |

### 1.4 命名規約

- 画面仕様書ファイル名：`<画面ID>_<画面名>.md`（例：`04-00-00_WBS一覧画面.md`）
- URL パス：`/app/<機能英語名>/<操作>`（例：`/app/wbs/list`、`/app/tasks/create`）。詳細は詳細設計の各画面仕様書で確定する。

## 5. 共通画面要素

すべての認証済み画面に共通する要素。

| 要素 | 内容 |
|------|------|
| グローバルヘッダ | ロゴ・プロジェクト切り替え・通知・ユーザーメニュー |
| サイドナビ | ダッシュボード／WBS／ガント／ロードマップ／レビュー／報告書／テンプレート／設定 |
| ブレッドクラム | 現在位置表示 |
| フラッシュメッセージ | 画面上部にトースト形式で成功／エラーメッセージを表示 |
| 権限不足時 | `403 アクセス権がありません` 画面へリダイレクト |
| 未認証時 | `01-01-04 ログイン画面` へリダイレクト（遷移後に元画面へ復帰） |
