# 【API設計書】ログアウトAPI

## 1. 概要

リフレッシュトークンをブラックリストに追加して無効化する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/logout/`
- **認証:** 必要（Bearer トークン）

## 3. リクエスト

### ボディ（JSON）

#### `refresh`

- **型:** `string` / **必須:** ✅
- 無効化するリフレッシュトークン

```json
{
  "refresh": "<JWT refresh token>"
}
```

## 4. レスポンス

### 成功 (205 Reset Content)

ボディなし

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| トークン無効・期限切れ | 400 | トークンが無効です |
| 認証なし | 401 | 認証情報がありません |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - アクセストークンの検証（Authorization ヘッダー）
2. **ブラックリスト登録**
   - simplejwt の `TokenBlacklist` に `refresh` トークンを追加
   - 失敗（トークン無効・期限切れ）：400 を返す
3. **レスポンス**
   - 205 Reset Content を返す
