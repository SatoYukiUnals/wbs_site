# 【API設計書】ユーザー削除

## 1. 概要

同テナント内の指定ユーザーを削除する。マスターユーザーのみ実行可能。自分自身の削除は不可。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/auth/users/{user_id}/`
- **認証:** JWT必須（masterのみ）

## 3. リクエスト

### パスパラメータ

#### `user_id`（ユーザーID）

- **型:** `integer` / **必須:** ○

## 4. レスポンス

### 成功 (204 No Content)

レスポンスボディなし

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 自分自身の削除 | 403 | `ERR_PERM_003` | 自分自身は削除できません | |
| 対象ユーザー未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限チェック**
   - `role` が `master` であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **自己削除チェック**
   - `user_id` が JWT の `user_id` と一致しないことを確認
     - 一致：403 `ERR_PERM_003` を返す
3. **対象ユーザー取得**
   - `User` テーブルから `user_id` AND `tenant_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
4. **DB削除**
   - `User` レコードを DELETE
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 204 No Content を返却
