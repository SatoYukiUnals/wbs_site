# 【画面仕様書】タスク作成画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 04-01-01 |
| 画面名 | タスク作成画面 |
| URL | `/projects/{id}/tasks/new` |
| 対応デザイン | - |

## 2. 目的

プロジェクトに新しいタスクを作成する。admin以上のみ操作可能。

## 3. 処理概要

### 画面で操作できる情報

- タスク名・説明・開始日・終了日・ステータス・親タスク・担当者の入力
- タスクの新規作成

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| WBS一覧画面 | 新規作成ボタン | タスク作成画面 |
| タスク作成画面 | 作成成功 | WBS一覧画面 |
| タスク作成画面 | キャンセルボタン | WBS一覧画面 |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| タスク名 | submit | 必須・1〜200文字 | ERR_VAL_001 | タスク名を入力してください |
| 終了日 | submit | 開始日以降（両方入力時） | ERR_VAL_001 | 終了日は開始日以降の日付を入力してください |
| 親タスク | submit | 5階層を超えないこと | ERR_VAL_001 | 階層は最大5階層までです |

## 8. アクション

### 画面表示時

1. `GET /api/v1/projects/{id}/tasks/` で親タスク候補を取得
2. `GET /api/v1/projects/{id}/members/` で担当者候補を取得

### 作成ボタン押下時

1. フロント側バリデーション実行
2. `POST /api/v1/projects/{id}/tasks/` 呼び出し
3. 成功（201）: `/projects/{id}/wbs` へ遷移
4. 失敗（400）: エラーメッセージを表示
