# 【API設計書】自動割り振り確定

## 1. 概要

プレビューで確認した自動割り振り結果を確定してDBに保存する。admin 以上のみ実行可能。割り振り履歴を記録する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/auto-assign/confirm/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `assignments`（割り振り内容）

- **型:** `array` / **必須:** ○
- 1件以上

各要素：

| フィールド | 型 | 必須 | 備考 |
| :--- | :--- | :--- | :--- |
| `task_id` | integer | ○ | 同プロジェクト内のタスクID |
| `user_id` | integer | ○ | プロジェクトメンバーのユーザーID |

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "log_id": 5,
  "assigned_count": 8,
  "executed_at": "2026-04-17T10:00:00Z"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - `assignments` が1件以上であることを確認
   - 各 `task_id` が同プロジェクト内に存在することを確認
   - 各 `user_id` がプロジェクトメンバーであることを確認
     - エラー時：400 `ERR_VAL_001` を返す
3. **DB操作（トランザクション）**
   - 各タスクの `TaskAssignee` を INSERT（既存の担当者は上書き）
   - `AutoAssignLog` を INSERT（`result` = assignments の JSON・`executed_by` = `user_id`）
     - DBエラー：500 `ERR_SYS_500` を返す
4. **レスポンス生成**
   - 作成した log_id・割り当て件数・実行日時を返却
