# 【API設計書】トークンリフレッシュ

## 1. 概要

有効なリフレッシュトークンを受け取り、新しいアクセストークンを発行する。アクセストークンの有効期限（15分）切れ時にクライアントが呼び出す。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/token/refresh/`
- **認証:** 不要

## 3. リクエスト

### ボディ

#### `refresh_token`（リフレッシュトークン）

- **型:** `string` / **必須:** ○

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "access_token": "eyJ..."
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| トークン無効・期限切れ | 401 | `ERR_AUTH_003` | トークンが無効または期限切れです | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **バリデーション**
   - `refresh_token` の存在チェック
     - エラー時：400 `ERR_VAL_001` を返す
2. **リフレッシュトークン検証**
   - JWT署名・有効期限チェック
   - Redis に該当トークンが存在するか確認（ログアウト済み無効化チェック）
     - 無効 or 期限切れ or Redis未存在：401 `ERR_AUTH_003` を返す
3. **アクセストークン発行**
   - リフレッシュトークンのペイロード（`user_id`・`tenant_id`・`role`）を引き継ぎ
   - 新しいアクセストークン（15分）を発行
4. **レスポンス生成**
   - 新しいアクセストークンを返却
