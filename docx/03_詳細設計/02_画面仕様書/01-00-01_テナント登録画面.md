# 【画面仕様書】テナント登録画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 01-00-01 |
| 画面名 | テナント登録画面 |
| URL | `/register` |
| 対応デザイン | - |

## 2. 目的

新規テナント（組織）とマスターユーザーを同時に登録する。

## 3. 処理概要

### 画面で操作できる情報

- テナント名の入力
- マスターユーザー情報（メールアドレス・パスワード・ユーザー名）の入力
- テナント登録の実行

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| - | 直接アクセス | テナント登録画面 |
| テナント登録画面 | 登録成功 | ログイン画面 |
| テナント登録画面 | ログインリンク押下 | ログイン画面 |

## 4. アクセス権限

| :--- | :--- |
| 未ログイン | ○ |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| テナント名 | submit | 必須・1〜100文字 | ERR_VAL_001 | テナント名を入力してください |
| ユーザー名 | submit | 必須・1〜50文字 | ERR_VAL_001 | ユーザー名を入力してください |
| メールアドレス | submit | 必須・メール形式 | ERR_VAL_001 | 有効なメールアドレスを入力してください |
| パスワード | submit | 必須・8文字以上 | ERR_VAL_001 | パスワードは8文字以上で入力してください |
| パスワード（確認） | submit | パスワードと一致 | ERR_VAL_001 | パスワードが一致しません |

## 8. アクション

### 画面表示時

- ログイン済みの場合、`/dashboard` へリダイレクト

### 登録ボタン押下時

1. フロント側バリデーション実行
2. `POST /api/v1/auth/register-tenant/` 呼び出し
3. 成功（201）: 成功メッセージ表示後、`/login` へ遷移
4. 失敗（400）: エラーメッセージをフィールド横に表示
