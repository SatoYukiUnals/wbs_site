# 【画面仕様書】ログイン画面

## 1. 基本情報

| 項目 | 内容 |

| 画面ID | 01-01-04 |

## 2. 目的

登録済みユーザーがメールアドレスとパスワードでログインし、JWTトークンを取得する。

## 3. 処理概要

### 画面で操作できる情報

- メールアドレス・パスワードの入力
- ログインの実行

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| - | 直接アクセス | ログイン画面 |
| テナント登録画面 | 登録成功 | ログイン画面 |
| ログイン画面 | ログイン成功 | ダッシュボード画面 |
| ログイン画面 | テナント登録リンク押下 | テナント登録画面 |

## 4. アクセス権限

| :--- | :--- |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| メールアドレス | submit | 必須・メール形式 | ERR_VAL_001 | メールアドレスを入力してください |
| パスワード | submit | 必須 | ERR_VAL_001 | パスワードを入力してください |
| 認証失敗 | APIレスポンス | 401 | ERR_AUTH_001 | メールアドレスまたはパスワードが正しくありません |

## 8. アクション

### 画面表示時

- ログイン済みの場合、`/dashboard` へリダイレクト

### ログインボタン押下時

1. フロント側バリデーション実行
2. `POST /api/v1/auth/login/` 呼び出し
3. 成功（200）: `access_token`・`refresh_token` をストアに保存し、`/dashboard` へ遷移
4. 失敗（401）: 認証エラーメッセージを表示
