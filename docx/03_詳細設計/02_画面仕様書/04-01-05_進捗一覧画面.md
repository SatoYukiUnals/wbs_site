## 目的

プロジェクトのタスク構造を3階層に集計し、時間・件数・進捗率・遅延状況を一覧で把握する。

## 機能仕様

### ■ 集計ロジック

#### 対象階層

depth 0 / 1 / 2 の3階層を集計する。

#### ステータス判定ルール

`status` フィールドは使用せず、実績日付でステータスを判定する。

| 判定結果 | 条件 |
|---------|------|
| Done | `actual_end_date` が入力済み |
| InProgress | `actual_start_date` が入力済みかつ `actual_end_date` が NULL |
| Todo | `actual_start_date` が NULL |

#### スキップ条件

同名の親ノードかつ実タスク（実際の作業行）を持たないノードはスキップする。

#### 遅延/巻き計算ルール

| 表示 | 条件 |
|------|------|
| マイナス（赤） | `end_date` ≤ 今日 かつ 未完了（Done でない） |
| プラス（青） | `end_date` > 今日 かつ 全タスク完了（Done） |
| 表示なし | 上記以外 |

### ■ 項目一覧

| 項目名 | 内容 | 備考 |
|--------|------|------|
| 項番 | 階層に応じたインデント付き番号 | |
| タイトル | インデント付きタスク名 | |
| 総時間(h) | `estimated_hours` の合計 | |
| Done(h) | Doneタスクの `estimated_hours` 合計 | |
| 進行(h) | InProgressタスクの `estimated_hours` 合計 | |
| 未着手(h) | Todoタスクの `estimated_hours` 合計 | |
| 進捗(%) | Done(h) / 総時間(h) × 100 | |
| 総件数 | タスク件数の合計 | |
| Done(件) | Doneタスクの件数 | |
| 進行(件) | InProgressタスクの件数 | |
| 未着手(件) | Todoタスクの件数 | |
| 進捗(件数) | Done(件) / 総件数 × 100（%表示） | |
| 予定開始日 | 配下タスクの最早 `start_date` | |
| 予定終了日 | 配下タスクの最遅 `end_date` | |
| 実績開始日 | 配下タスクの最早 `actual_start_date` | |
| 実績終了日 | 配下タスクの最遅 `actual_end_date` | |
| 遅延/巻き(h) | 遅延はマイナス（赤）・巻きはプラス（青）で表示 | 計算ルールは上記参照 |

### ■ レイアウト

- 横スクロール・縦スクロール対応（列数が多いため横スクロールを標準とする）
- ヘッダー行は縦スクロール時に固定表示する。
