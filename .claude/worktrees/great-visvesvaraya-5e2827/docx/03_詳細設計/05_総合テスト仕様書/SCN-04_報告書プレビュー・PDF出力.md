# 【総合テスト：シナリオ】SCN-04 報告書プレビュー・PDF出力

## 1. シナリオ概要

- **目的**：報告書プレビューのデータ整合性を確認し、PDF出力が正しい内容・形式で生成されることを検証する。
- **前提条件**：SCN-02完了済み。タスクに完了・進行中・期限超過が混在している。
- **実行ユーザー**：adminユーザー

---

## 2. テストシナリオ

### Step 1：報告書作成画面を開く

- **操作**：報告書作成画面を開き、対象期間（例：2026-04-01〜2026-06-30）を入力してプレビューボタンを押す。
- **確認**：
  - [ ] `GET /api/v1/projects/{id}/reports/generate/?start_date=2026-04-01&end_date=2026-06-30` が呼ばれる。
  - [ ] `200 OK` でJSONレスポンスが返却される。

---

### Step 2：プレビュー集計値の検証

- **操作**：レスポンスの集計値を確認する。
- **確認**：
  - [ ] `project_name` がプロジェクト名と一致する。
  - [ ] `total_tasks` が対象期間内のタスク件数と一致する。
  - [ ] `completed_tasks` が `status=completed` の件数と一致する。
  - [ ] `overdue_tasks` が `end_date < 本日` かつ未完了の件数と一致する。
  - [ ] `overall_progress` が対象タスクの `progress_rate` 平均値（小数点以下切り捨て）と一致する。

---

### Step 3：メンバー別進捗の検証

- **操作**：レスポンスの `member_stats` を確認する。
- **確認**：
  - [ ] 全プロジェクトメンバーが `member_stats` に含まれる。
  - [ ] 各メンバーの `assigned_count`、`completed_count` がDB値と一致する。

---

### Step 4：PDF出力ボタンを押す

- **操作**：プレビュー確認後、PDF出力ボタンを押す。
- **確認**：
  - [ ] `POST /api/v1/projects/{id}/reports/export-pdf/` が呼ばれる。
  - [ ] ファイル名が `Report_{プロジェクト名}_{YYYYMMDD}.pdf`。
  - [ ] Content-Type が `application/pdf`。
  - [ ] ファイルサイズが 0 より大きい（正常生成）。

---

### Step 5：PDFの内容確認

- **操作**：ダウンロードしたPDFを開く。
- **確認**：
  - [ ] 表紙にプロジェクト名・報告期間・出力日・作成者名が表示される。
  - [ ] セクション2にプログレスバーと集計値が表示される。
  - [ ] セクション3のタスク一覧が `order` 昇順で表示される。
  - [ ] 期限超過タスクのテキストが赤色で表示される。
  - [ ] セクション4のメンバー別進捗にユーザー名・担当数・完了数・平均進捗率が表示される。
  - [ ] A4縦でフッターにページ番号（例：`1 / 3`）が表示される。

---

### Step 6：タスク0件時のPDF出力

- **操作**：対象期間にタスクが存在しない条件でPDF出力する。
- **確認**：
  - [ ] PDFが生成される（エラーにならない）。
  - [ ] セクション3に「対象期間内にタスクはありません」と表示される。

---

### Step 7：プレビューとPDFのデータ一致確認

- **確認**：
  - [ ] Step2で確認した集計値とPDFの集計値が一致する。
  - [ ] プレビューのタスク一覧とPDFのセクション3のタスク一覧が一致する。
