# 【API設計書】プロフィール取得

## 1. 概要

ログイン中ユーザー自身のプロフィール情報を返す。JWTの `user_id` を使って対象ユーザーを特定する。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/auth/profile/`
- **認証:** JWT必須（全ユーザー）

## 3. リクエスト

### クエリパラメータ

なし

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "username": "hanako",
  "email": "hanako@example.com",
  "role": "admin",
  "tenant_id": 1,
  "created_at": "2026-04-01T00:00:00Z"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `user_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **データ取得**
   - `User` テーブルから `user_id` で SELECT
     - DBエラー：500 `ERR_SYS_500` を返す
3. **レスポンス生成**
   - ユーザー情報を返却
