# 【API設計書】ロードマップアイテム作成

## 1. 概要

指定プロジェクトのロードマップにアイテムを新規作成する。admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/roadmap/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `title`（タイトル）

- **型:** `string` / **必須:** ○
- 1〜200文字

#### `description`（説明）

- **型:** `string` / **必須:** -

#### `quarter_id`（クォーターID）

- **型:** `integer` / **必須:** ○
- 同プロジェクト内のクォーターIDであること

#### `status`（ステータス）

- **型:** `string` / **必須:** -
- `未着手` / `進行中` / `完了` のいずれか。省略時は `未着手`

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "id": 3,
  "title": "認証機能リリース",
  "description": "JWT認証の実装",
  "quarter_id": 1,
  "status": "未着手",
  "project_id": 1
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - `title`・`quarter_id` の存在チェック
   - `quarter_id` が同プロジェクト内に存在することを確認
   - `status` の値チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **DB操作**
   - `RoadmapItem` レコードを INSERT
     - DBエラー：500 `ERR_SYS_500` を返す
4. **レスポンス生成**
   - 作成したアイテム情報を返却
