# 【画面仕様書】タスク編集画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 04-01-02 |
| 画面名 | タスク編集画面 |
| URL | `/projects/{id}/tasks/{task_id}/edit` |
| 対応デザイン | - |

## 2. 目的

既存タスクの情報を編集する。admin以上は全項目を編集可能。担当者はステータス・進捗率のみ変更可能。

## 3. 処理概要

### 画面で操作できる情報

- タスク名・説明・開始日・終了日・ステータス・進捗率・担当者の編集
- 権限によって編集可能フィールドが異なる

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| WBS一覧画面 | 編集ボタン | タスク編集画面 |
| タスク詳細画面 | 編集ボタン | タスク編集画面 |
| タスク編集画面 | 保存成功 | WBS一覧画面 |
| タスク編集画面 | キャンセルボタン | WBS一覧画面 |

## 4. アクセス権限

| ロール | アクセス可否 | 備考 |
| :--- | :--- | :--- |
| master / admin | ○ | 全項目編集可 |
| member（担当者） | ○ | ステータス・進捗率のみ |
| member（非担当者） | - | |
| 未ログイン | - | |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| タスク名 | submit | 必須・1〜200文字 | ERR_VAL_001 | タスク名を入力してください |
| 進捗率 | submit | 0〜100の整数 | ERR_VAL_001 | 進捗率は0〜100で入力してください |
| 終了日 | submit | 開始日以降（両方入力時） | ERR_VAL_001 | 終了日は開始日以降の日付を入力してください |

## 8. アクション

### 画面表示時

1. `GET /api/v1/projects/{id}/tasks/{task_id}/` 呼び出し
2. 取得データをフォームに設定
3. ロールに応じて編集可能フィールドを制御

### 保存ボタン押下時

1. フロント側バリデーション実行
2. API呼び出し（PUT or PATCH）
3. 成功: `/projects/{id}/wbs` へ遷移
