# 【総合テスト：テスト項目一覧】（対象：レビュー差し戻し・再レビューフロー）

## 1. シナリオ概要

レビュアーが差し戻し（指摘コメント付き）→ 対応者が対応完了を報告 → 確認待ちに変更 → 指摘者が確認して完了 という差し戻しループを E2E で検証する。

---

## 2. 実施環境

- **OS**：Windows 11 / macOS
- **ブラウザ**：Google Chrome（最新版）
- **先行シナリオ**：02_タスク進捗管理フロー（No.3 レビュー依頼まで完了している状態）または Case 20 の事前データ使用

---

## 3. テスト項目一覧

### No.1：差し戻し（指摘コメント2件）

- **前提条件：** Case 20（レビュー待ちタスク）
- **操作者：** adminユーザー（レビュアー）
- **テスト内容：**
  1. Case 20 のレビュー詳細画面を開く
  2. 指摘コメントを2件入力する（「コメントA」「コメントB」）
  3. 「差し戻し」ボタンを押下する
- **期待結果：**
  - [ ] 画面：レビューステータスが「rejected」に表示されること
  - [ ] 画面：タスクのステータスが「進行中」に戻っていること
  - [ ] 画面：コメント一覧に「コメントA」「コメントB」が表示されること
  - [ ] 画面：操作履歴に「reject」アクションと実行者・日時が表示されること
- **事後状態：** タスク=進行中、review=rejected、コメント2件

### No.2：対応者によるコメント編集・確認待ちに変更

- **前提条件：** No.1 完了後
- **操作者：** memberユーザー（担当者・対応者）
- **テスト内容：**
  1. レビュー詳細画面を開き「コメントA」を「コメントA（対応済み）」に編集する
  2. レビューステータスを「確認待ち」に変更する
- **期待結果：**
  - [ ] 画面：コメントA の本文が「コメントA（対応済み）」に更新されること
  - [ ] 画面：レビューステータスが「確認待ち」に表示されること
  - [ ] 画面：操作履歴に「status_change（確認待ち）」アクションが追加されること
- **事後状態：** review=確認待ち

### No.3：指摘者による確認完了

- **前提条件：** No.2 完了後
- **操作者：** adminユーザー（元の指摘者）
- **テスト内容：**
  1. レビュー詳細画面で「完了」ステータスに変更する
  2. 確認ダイアログで「完了」を選択する
- **期待結果：**
  - [ ] 画面：レビューステータスが「完了」に表示されること
  - [ ] 画面：コメント編集フィールドが非活性になること
  - [ ] 画面：ステータス変更ボタンが非表示になること（完了ロック）
  - [ ] 画面：操作履歴に「complete」アクションが追加されること
- **事後状態：** review=完了（ロック済み）
