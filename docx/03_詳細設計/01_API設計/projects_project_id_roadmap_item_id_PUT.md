# 【API設計書】ロードマップアイテム編集

## 1. 概要

指定ロードマップアイテムのタイトル・説明・クォーター・ステータスを更新する。admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/projects/{project_id}/roadmap/{item_id}/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

#### `item_id`（アイテムID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `title`（タイトル）

- **型:** `string` / **必須:** ○
- 1〜200文字

#### `description`（説明）

- **型:** `string` / **必須:** -

#### `quarter_id`（クォーターID）

- **型:** `integer` / **必須:** ○
- 同プロジェクト内のクォーターIDであること

#### `status`（ステータス）

- **型:** `string` / **必須:** -
- `未着手` / `進行中` / `完了` のいずれか

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "title": "認証機能リリース 更新",
  "description": "更新後の説明",
  "quarter_id": 2,
  "status": "完了"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **対象アイテム確認**
   - `RoadmapItem` を `item_id` AND `project_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
3. **バリデーション**
   - `title`・`quarter_id` の存在チェック
   - `quarter_id` が同プロジェクト内に存在することを確認
     - エラー時：400 `ERR_VAL_001` を返す
4. **DB更新**
   - `RoadmapItem` レコードを UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 更新後のアイテム情報を返却
