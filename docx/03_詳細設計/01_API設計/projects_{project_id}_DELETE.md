# 【API設計書】プロジェクト削除API

## 1. 概要

プロジェクトを論理削除する（`deleted_at` にタイムスタンプを付与）。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/projects/{project_id}/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクト内 admin 以上

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `project_id` | UUID | ✅ | 対象プロジェクトID |

## 4. レスポンス

### 成功 (204 No Content)

ボディなし

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| プロジェクト存在しない | 404 | プロジェクトが見つかりません |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - プロジェクトメンバーかつ `role=admin` 以上。それ以外：403 を返す
2. **DB更新**
   - `project.deleted_at` に現在時刻を UPDATE
3. **レスポンス**
   - 204 No Content を返す
