# 【API設計書】メンバーロール変更API

## 1. 概要

プロジェクトメンバーのロールを変更する。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/projects/{project_id}/members/{user_id}/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクト内 admin 以上

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `project_id` | UUID | ✅ | 対象プロジェクトID |
| `user_id` | UUID | ✅ | 変更対象ユーザーID |

### ボディ（JSON）

```json
{ "role": "admin" }
```

## 4. レスポンス

### 成功 (200 OK)

```json
{ "user_id": "uuid", "role": "admin" }
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 対象メンバーが存在しない | 404 | 対象メンバーが見つかりません |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック** → admin 以上か確認
2. **メンバー検索** → `project_member` テーブルで `project_id` + `user_id` 検索。なければ 404
3. **DB更新** → `project_member.role` を UPDATE
4. **レスポンス** → 更新後情報を返す
