# 【API設計書】クォーター作成

## 1. 概要

指定プロジェクトにクォーターを新規作成する。admin 以上のみ実行可能。同プロジェクト内でのタイトル重複・期間重複はエラーとする。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/quarters/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `title`（クォータータイトル）

- **型:** `string` / **必須:** ○
- 例：Q1・Q2 など。同プロジェクト内で一意

#### `start_date`（開始日）

- **型:** `date` / **必須:** ○
- YYYY-MM-DD

#### `end_date`（終了日）

- **型:** `date` / **必須:** ○
- YYYY-MM-DD。`start_date` 以降であること

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "id": 3,
  "title": "Q3",
  "start_date": "2026-07-01",
  "end_date": "2026-09-30",
  "progress": 0,
  "project_id": 1
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| タイトル重複 | 400 | `ERR_VAL_007` | このタイトルは既に使用されています | 同プロジェクト内 |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - 全必須項目の存在チェック
   - `start_date` ≤ `end_date` チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **タイトル重複チェック**
   - 同 `project_id` 内で `title` が未使用であることを確認
     - 重複：400 `ERR_VAL_007` を返す
4. **DB操作**
   - `Quarter` レコードを INSERT（`progress` = 0）
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 作成したクォーター情報を返却
