# 【画面仕様書】メンバー管理画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 02-01-00 |
| 画面名 | メンバー管理画面 |
| URL | `/projects/{id}/members` |
| 対応デザイン | - |

## 2. 目的

プロジェクトのメンバー一覧を管理し、メンバーの追加・ロール変更・削除を行う。

## 3. 処理概要

### 画面で操作できる情報

- プロジェクトメンバー一覧の表示
- メンバーの追加
- メンバーのロール変更
- メンバーの削除

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| プロジェクト詳細画面 | メンバー管理リンク | メンバー管理画面 |
| メンバー管理画面 | 戻るボタン | プロジェクト詳細画面 |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |

| ロール（追加時） | submit | admin / member のいずれか | ERR_VAL_001 | ロールを選択してください |

## 8. アクション

### 画面表示時

1. `GET /api/v1/projects/{id}/members/` 呼び出し
2. メンバー一覧を表示

### メンバー追加ダイアログ「追加」押下時

1. `POST /api/v1/projects/{id}/members/` 呼び出し
2. 成功（201）: 一覧を再取得して表示

### ロール変更時

1. `PUT /api/v1/projects/{id}/members/{user_id}/` 呼び出し
2. 成功（200）: 表示を更新

### 削除確認「削除」押下時

1. `DELETE /api/v1/projects/{id}/members/{user_id}/` 呼び出し
2. 成功（204）: 一覧を再取得して表示
