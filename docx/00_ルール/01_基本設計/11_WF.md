# 基本設計：WF（画面レイアウト）

## 1. ■概要・目的

各画面に表示されるUI要素（フォーム・ボタン・テーブル・表示項目等）を明確にするためのドキュメントです。操作フロー・遷移条件・表示制御は機能仕様書に記載します。

## 3. ■作業手順（Issueのタスクリストとして利用）

- [ ] **画面一覧の整理**: 対象機能で使用される画面を洗い出します。
- [ ] **画面要素の定義**: 各画面に表示されるフォーム・ボタン・テーブル等のUI要素を列挙します。
- [ ] **機能仕様書への反映**: 画面要素をもとに、機能仕様書のワークフロー・処理フローに反映します。

## 4. ■セルフチェック項目（作業完了後の最終確認）
