# 【API設計書】ログアウト

## 1. 概要

リフレッシュトークンをRedisから削除してセッションを無効化する。アクセストークンは有効期限（15分）切れまで有効のため、クライアント側でも破棄する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/logout/`
- **認証:** JWT必須（全ユーザー）

## 3. リクエスト

### ボディ

#### `refresh_token`（リフレッシュトークン）

- **型:** `string` / **必須:** ○

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "message": "ログアウトしました"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| トークン無効 | 400 | `ERR_VAL_004` | トークンが無効です | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - AuthorizationヘッダーのアクセストークンをJWT検証
     - 無効：401 `ERR_AUTH_002` を返す
2. **リフレッシュトークン検証**
   - `refresh_token` の存在チェック・形式チェック
     - エラー時：400 `ERR_VAL_004` を返す
3. **Redis削除**
   - `refresh_token` に対応するRedisキーを削除（無効化）
4. **レスポンス生成**
   - 成功メッセージを返却
