# 【画面仕様書】ユーザー一覧画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 01-03-00 |
| 画面名 | ユーザー一覧画面 |
| URL | `/users` |
| 対応デザイン | - |

## 2. 目的

テナント内の全ユーザーを一覧表示し、ユーザーの作成・編集・削除を行う。

## 3. 処理概要

### 画面で操作できる情報

- テナント内ユーザーの一覧表示
- ユーザー作成画面への遷移
- ユーザーロール・ステータスの変更
- ユーザーの削除

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| ヘッダーメニュー | ユーザー管理クリック | ユーザー一覧画面 |
| ユーザー一覧画面 | 新規作成ボタン | ユーザー作成画面 |
| ユーザー一覧画面 | 編集ボタン | ユーザー作成画面（編集モード）→ 実際はモーダルまたはインライン |

なし

## 8. アクション

### 画面表示時

1. `GET /api/v1/auth/users/` 呼び出し
2. ユーザー一覧をストアに格納して表示

### 削除確認ダイアログ「削除」押下時

1. `DELETE /api/v1/auth/users/{user_id}/` 呼び出し
2. 成功（204）: 一覧を再取得して表示を更新
3. 失敗（403）: エラーメッセージを表示
