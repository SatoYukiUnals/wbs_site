# 【API設計書】ユーザー情報・ロール変更

## 1. 概要

同テナント内の指定ユーザーの情報（ユーザー名・メールアドレス）およびロールを変更する。マスターユーザーのみ実行可能。自分自身のロール変更は不可。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/auth/users/{user_id}/`
- **認証:** JWT必須（masterのみ）

## 3. リクエスト

### パスパラメータ

#### `user_id`（ユーザーID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `username`（ユーザー名）

- **型:** `string` / **必須:** -
- 1〜150文字

#### `email`（メールアドレス）

- **型:** `string` / **必須:** -
- メール形式。全テナント横断で一意

#### `role`（ロール）

- **型:** `string` / **必須:** -
- `admin` または `member` のいずれか

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 2,
  "username": "taro_updated",
  "email": "taro_new@example.com",
  "role": "member",
  "tenant_id": 1
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 自分自身のロール変更 | 403 | `ERR_PERM_002` | 自分自身のロールは変更できません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| メールアドレス重複 | 400 | `ERR_VAL_003` | このメールアドレスは既に使用されています | |
| 対象ユーザー未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限チェック**
   - `role` が `master` であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **自己変更チェック**
   - `user_id` が JWT の `user_id` と一致しないことを確認
     - 一致：403 `ERR_PERM_002` を返す
3. **対象ユーザー取得**
   - `User` テーブルから `user_id` AND `tenant_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
4. **バリデーション**
   - 変更フィールドの形式チェック・`role` 値チェック
     - エラー時：400 `ERR_VAL_001` を返す
5. **メールアドレス重複チェック**（`email` 変更時のみ）
   - `User` テーブル全体で `email` 未使用を確認
     - 重複：400 `ERR_VAL_003` を返す
6. **DB更新**
   - 指定フィールドのみ UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
7. **レスポンス生成**
   - 更新後のユーザー情報を返却
