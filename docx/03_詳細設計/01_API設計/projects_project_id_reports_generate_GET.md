# 【API設計書】報告書内容自動生成

## 1. 概要

指定期間またはクォーターのデータを集計して報告書の各セクション（概要・進捗サマリー・クォーター別進捗・タスク一覧・遅延タスク・レビュー状況）をテキストとして生成する。DBへの保存は行わない。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/{project_id}/reports/generate/`
- **認証:** JWT必須（プロジェクトメンバー以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### クエリパラメータ

#### `period_start`（集計開始日）

- **型:** `date` / **必須:** △（`quarter_id` 未指定時は必須）
- YYYY-MM-DD

#### `period_end`（集計終了日）

- **型:** `date` / **必須:** △（`quarter_id` 未指定時は必須）
- YYYY-MM-DD。`period_start` 以降であること

#### `quarter_id`（クォーターID）

- **型:** `integer` / **必須:** △（`period_start`/`period_end` 未指定時は必須）

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "sections": {
    "overview": "新規システム開発 進捗報告 (2026-04-01 〜 2026-04-30)",
    "summary": "全体進捗率: 35%。完了タスク: 15件、進行中: 8件、遅延: 3件",
    "quarters": "Q1 進捗: 60%",
    "tasks": "タスク一覧:\n- 要件定義 (完了 100%)\n- 設計 (進行中 50%)",
    "delayed": "遅延タスク:\n- テスト実施 (期限: 2026-04-10)",
    "reviews": "差し戻し中: 1件、確認待ち: 2件"
  },
  "period": {
    "start": "2026-04-01",
    "end": "2026-04-30"
  }
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | 期間・クォーターのいずれも未指定 |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・メンバー確認**
   - `ProjectMember` でメンバーシップを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 非メンバー：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - `quarter_id` または `period_start`・`period_end` の組合せのいずれかが指定されていることを確認
   - `period_start` ≤ `period_end` チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **データ取得・集計**
   - `Task` を対象期間でフィルタして SELECT
   - `Quarter` を `project_id` でフィルタして SELECT
   - `Review` を `task_id` でフィルタして SELECT（差し戻し中・確認待ち）
     - DBエラー：500 `ERR_SYS_500` を返す
4. **テキスト生成**
   - 各セクション（overview・summary・quarters・tasks・delayed・reviews）のテキストを生成
5. **レスポンス生成**
   - 生成テキストを返却（DBに保存しない）
