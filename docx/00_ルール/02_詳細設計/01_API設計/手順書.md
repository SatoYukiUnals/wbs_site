# 詳細設計：API設計（Markdown形式）

## 1. ■概要・目的
実装時に参照する詳細なAPIインターフェースを定義します。入出力項目、データ型、バリデーションルール、エラーレスポンス等を具体化することで、Backend実装とFrontendのAPI呼出し処理に齟齬がないようにします。

## 2. ■事前準備
- [ ] 規定のテンプレート：[サンプル](./sample.md)
- [ ] 基本設計のAPI概要、テーブル定義の確認

## 3. ■作業手順（Issueのタスクリストとして利用）
- [ ] **テンプレートの利用**: 規定のサンプルをコピーして作成を開始します。
- [ ] **基本情報の定義**: API名、メソッド、エンドポイントを定義します。
- [ ] **リクエスト（入力値）の定義**: クエリパラメータ、ボディの項目名、型、必須/任意、バリデーションルールを詳細に記載します。
- [ ] **レスポンス（成功/失敗）の定義**: 200 OK時のJSON構造と、想定されるエラー（4xx, 5xx等）のステータスコード、エラー内容を具体化します。
- [ ] **内部ロジックの記述**: SQLの構成イメージや、ビジネスロジックのフロー、加工処理などを、実装者が迷わないレベルまで具体化します。エラーが発生する箇所は、各処理項目の下にインデントして記載します。

## 4. ■セルフチェック項目（作業完了後の最終確認）
- [ ] 規定のサンプルに基づき、Markdown形式で作成されている
- [ ] エンドポイント、入力（型・バリデーション）、出力（200・エラー）が明記されている
- [ ] 内部処理ロジックが実装可能な粒度で記載されている
- [ ] 処理の分岐に関して明示的に記載されていること
- [ ] 内部処理でエラーが発生する箇所がインデントで明示されている
- [ ] エラーメッセージをエラーメッセージ仕様書から使用していること
- [ ] AIによる自動生成内容を人間がレビューし、必要に応じて修正・補足されている
