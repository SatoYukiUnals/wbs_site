# 【API設計書】プロフィール更新API

## 1. 概要

ログイン中のユーザー自身の氏名・メールアドレスを更新する。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/auth/profile/`
- **認証:** 必要（Bearer トークン）

## 3. リクエスト

### ボディ（JSON）

#### `username`

- **型:** `string` / **必須:** ✅
- 1〜100文字

#### `email`

- **型:** `string` / **必須:** ✅
- RFC5322 形式。同テナント内で重複不可（自分自身は除く）

```json
{
  "username": "山田太郎",
  "email": "yamada@example.com"
}
```

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "user_id": "uuid",
  "username": "山田太郎",
  "email": "yamada@example.com"
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| email 重複 | 409 | このメールアドレスは既に使用されています |
| email 形式不正 | 400 | 正しいメールアドレスを入力してください |
| 認証なし | 401 | 認証情報がありません |

## 5. 内部処理 / ロジック

1. **バリデーション**
   - `email` 形式チェック・同テナント内重複チェック（自分以外）
   - 重複：409 を返す
2. **DB更新**
   - `user` テーブルの `username`・`email` を UPDATE
3. **レスポンス**
   - 更新後のユーザー情報を返す
