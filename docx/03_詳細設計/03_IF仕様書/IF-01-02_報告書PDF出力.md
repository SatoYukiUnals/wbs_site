# IF-01-02 報告書PDF出力

## 基本情報

| 項目 | 内容 |
| :--- | :--- |
| **IFID** | IF-01-02 |
| **IF名** | 報告書PDF出力 |
| **出力形式** | PDF |
| **文字コード** | UTF-8 |
| **実行タイミング** | 09-01-04 報告書作成・編集画面の「PDF出力」ボタン押下時 |

---

## 目的

09-01-04 画面で自動生成・手動編集した報告書の内容をPDFに変換してダウンロードする。DBへの保存は行わない（都度生成）。

---

## 対象データ

- フロントエンドが POST リクエストのボディとして送信した編集済み報告書セクション（DBからの直接取得はしない）

---

## 機能仕様

### ■ 項目一覧

| No | セクション名 | 物理名 | 型 | 繰り返し |
| :--- | :--- | :--- | :--- | :--- |
| 1 | プロジェクト概要 | sections.overview | string | - |
| 2 | 進捗サマリー | sections.summary | string | - |
| 3 | クォーター別進捗 | sections.quarters | string | - |
| 4 | タスク一覧 | sections.tasks | string | - |
| 5 | 遅延タスク | sections.delayed | string | - |
| 6 | レビュー状況 | sections.reviews | string | - |

#### ▶ 備考

| 項目名 | 備考 |
| :--- | :--- |
| 各セクション | テキスト（Markdown）形式で受け取り、PDF上では整形済みテキストとして出力する |

### ■ 書式（PDF出力）

| 項目名 | フォント | 文字サイズ | 太字 | 文字色 | 背景色 | セル幅 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| タイトル（報告書タイトル） | Noto Sans JP | 18 | ✅ | #1F3864 | - | - |
| セクション見出し | Noto Sans JP | 13 | ✅ | #2E75B6 | - | - |
| 本文 | Noto Sans JP | 10 | - | #333333 | - | - |
| フッター（ページ番号・出力日時） | Noto Sans JP | 8 | - | #888888 | - | - |

#### ▶ 備考

| 項目名 | 備考 |
| :--- | :--- |
| フォント | WeasyPrint 使用のため、日本語対応フォント（Noto Sans JP）をサーバーにインストールしておく |
| ページ番号 | フッター右に「ページ / 総ページ数」を自動付与 |
| 出力日時 | フッター左に「出力日時：YYYY/MM/DD HH:mm」を自動付与 |

### ■ 変換・抽出ロジック

| 項目名 | ロジック |
| :--- | :--- |
| セクション内容 | リクエストボディの各フィールドをそのままHTMLテンプレートに埋め込む |
| PDF生成 | Django ビュー側でHTMLを組み立て、WeasyPrint（または ReportLab）で PDF バイナリに変換 |
| レスポンス | `Content-Disposition: attachment; filename="..."` で PDF バイナリを返す |

## 参照テーブル

報告書PDF出力時は DB を参照しない（リクエストボディの内容のみを使用）。

自動生成 API（`GET /api/v1/projects/{project_id}/reports/generate/`）が参照するテーブルは以下の通り：

### ■ `task`

| 項目名 | カラム | データ型 |
| :--- | :--- | :--- |
| タスク名 | `title` | `VARCHAR` |
| ステータス | `status` | `VARCHAR` |
| 進捗率 | `progress` | `INTEGER` |
| 終了日 | `end_date` | `DATE` |

### ■ `quarter`

| 項目名 | カラム | データ型 |
| :--- | :--- | :--- |
| クォーター名 | `title` | `VARCHAR` |
| 進捗率 | `progress` | `INTEGER` |

### ■ `review`

| 項目名 | カラム | データ型 |
| :--- | :--- | :--- |
| レビューID | `id` | `UUID` |
| ステータス | `status` | `VARCHAR` |
| タスクID | `task_id` | `UUID` |

### ■ `project`

| 項目名 | カラム | データ型 |
| :--- | :--- | :--- |
| プロジェクト名 | `name` | `VARCHAR` |
| 開始日 | `start_date` | `DATE` |
| 終了日 | `end_date` | `DATE` |

---

## 備考

- 報告書の内容はDBに保存しない。PDFはリクエストのたびに都度生成する。
- セクション内容が空文字の場合、該当セクションをPDF上で省略する。
- ファイル名規則：`{プロジェクト名}_報告書_{出力日 YYYYMMDD}.pdf`
- WeasyPrint または ReportLab を使用。日本語表示のためにサーバー側への Noto Sans JP フォントインストールが必要。
