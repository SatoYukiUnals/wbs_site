# 【API設計書】プロジェクト編集

## 1. 概要

指定プロジェクトの基本情報（名称・説明・期間）を更新する。プロジェクト内の admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/projects/{project_id}/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `name`（プロジェクト名）

- **型:** `string` / **必須:** ○
- 1〜200文字

#### `description`（説明）

- **型:** `string` / **必須:** -

#### `start_date`（開始日）

- **型:** `date` / **必須:** -
- YYYY-MM-DD

#### `end_date`（終了日）

- **型:** `date` / **必須:** -
- YYYY-MM-DD。`start_date` 以降であること

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "name": "新規システム開発 v2",
  "description": "更新後の説明",
  "start_date": "2026-04-01",
  "end_date": "2026-12-31",
  "progress": 35
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `tenant_id`・`user_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **プロジェクト取得・メンバー権限確認**
   - `Project` を `project_id` AND `tenant_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
   - `ProjectMember` で `user_id` の `role` が `admin` 以上であることを確認
     - 権限不足：403 `ERR_PERM_001` を返す
3. **バリデーション**
   - `name` の存在チェック
   - `start_date` ≤ `end_date` チェック
     - エラー時：400 `ERR_VAL_001` を返す
4. **DB更新**
   - `Project` レコードを UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 更新後のプロジェクト情報を返却
