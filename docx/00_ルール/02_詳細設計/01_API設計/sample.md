## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/orders/search/`

## 3. リクエスト

### クエリパラメータ / ボディ

#### `order_no`（注文番号）

- **型:** `string` / **必須:** ○
- 12文字。正規表現: `^[A-Z0-9-]{12}$`

#### `status`（ステータス）

- **型:** `string` / **必須:** -
- 共通区分 `ORDER_STATUS` から選択

#### `start_date`（検索開始日）

- **型:** `date` / **必須:** -
- YYYY-MM-DD。検索終了日以前であること

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "total_count": 1,
  "orders": [
    {
      "order_no": "ORD-2024-001",
      "status": "ordered",
      "order_date": "2024-04-01"
    }
  ]
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | メッセージ仕様書参照 |
| 該当データなし | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **バリデーションチェック**
    - `order_no` が指定されている場合、形式チェックを行う。
    - `start_date` と `end_date` の前後関係をチェックする。
        - バリデーションエラー：400 `ERR_VAL_001` を返す。
2. **分岐：権限チェック**
    - ログインユーザーが `Admin` の場合：全件検索対象とする。
    - それ以外の場合：自身の所属部門（`department_id`）に紐づくデータのみを検索対象とする。
3. **データ取得**
    - `orders` テーブルを主軸に、`customers` テーブルをJOINして情報を取得する。
    - 指定された検索条件、および権限によるフィルタを適用する。
        - 該当データなし：404 `ERR_DAT_404` を返す。
        - DBエラー：500 `ERR_SYS_500` を返す。
4. **レスポンス生成**
    - 取得結果をJSON形式に整形して返却する。
        - 予期しない例外：500 `ERR_SYS_500` を返す。
