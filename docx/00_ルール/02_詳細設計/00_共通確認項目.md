# 詳細設計：共通確認項目

## 1. ■概要・目的
詳細設計フェーズで作成された各ドキュメントの間で情報の矛盾がないか、また基本設計書の内容や共通ルールに準拠しているかを最終確認します。

## 2. ■セルフチェック項目（作業完了後の最終確認）
- [ ] 用語が用語集・共通ルールに準拠している
- [ ] 基本設計書（画面WF、処理フロー、テーブル定義等）との整合が取れている
- [ ] セキュリティ要件（SQLi、XSS、認可制御等）が「どう実装するか」のレベルまで具体化されている
- [ ] 第三者が読んで実装とユニットテストの着手が可能な状態である
- [ ] 不要な項目、および基本設計から変更のない重複記載を避けている
- [ ] AIによる自動生成内容を人間がレビューし、必要に応じて修正・補足されている
- [ ] ユーザー（角田さん）、PMが見てロジックや実装方針を理解できる内容になっている
