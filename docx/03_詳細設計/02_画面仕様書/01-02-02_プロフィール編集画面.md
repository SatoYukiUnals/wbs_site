# 【画面仕様書】プロフィール編集画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 01-02-02 |
| 画面名 | プロフィール編集画面 |
| URL | `/profile/edit` |
| 対応デザイン | - |

## 2. 目的

ログイン中ユーザーが自分のユーザー名・メールアドレス・パスワードを変更する。

## 3. 処理概要

### 画面で操作できる情報

- ユーザー名・メールアドレスの編集
- パスワードの変更

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| ヘッダーメニュー | プロフィール編集クリック | プロフィール編集画面 |
| プロフィール編集画面 | 保存成功 | プロフィール編集画面（完了メッセージ表示） |
| プロフィール編集画面 | キャンセルボタン | 直前の画面 |

## 4. アクセス権限

| ロール | アクセス可否 |
| :--- | :--- |
| 全ユーザー（ログイン済み） | ○ |

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| ユーザー名 | submit | 必須・1〜50文字 | ERR_VAL_001 | ユーザー名を入力してください |
| メールアドレス | submit | 必須・メール形式 | ERR_VAL_001 | 有効なメールアドレスを入力してください |
| 現在のパスワード | submit | 新しいパスワード入力時は必須 | ERR_VAL_001 | 現在のパスワードを入力してください |
| 新しいパスワード | submit | 入力時は8文字以上 | ERR_VAL_001 | パスワードは8文字以上で入力してください |
| 新しいパスワード（確認） | submit | 新しいパスワードと一致 | ERR_VAL_001 | パスワードが一致しません |

## 8. アクション

### 画面表示時

1. `GET /api/v1/auth/profile/` 呼び出し
2. 取得したユーザー名・メールアドレスをフォームに設定

### 保存ボタン押下時

1. フロント側バリデーション実行
2. `PUT /api/v1/auth/profile/` 呼び出し
3. 成功（200）: 「保存しました」トーストを表示
4. 失敗（400）: エラーメッセージを表示
