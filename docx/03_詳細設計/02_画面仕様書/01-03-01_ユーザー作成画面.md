# 【画面仕様書】ユーザー作成画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 01-03-01 |
| 画面名 | ユーザー作成画面 |
| URL | `/users/new` |
| 対応デザイン | - |

## 2. 目的

テナント内に新しいユーザーを作成する。マスターユーザーのみ操作可能。

## 3. 処理概要

### 画面で操作できる情報

- ユーザー名・メールアドレス・パスワード・ロールの入力
- ユーザーの新規作成

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| ユーザー一覧画面 | 新規作成ボタン | ユーザー作成画面 |
| ユーザー作成画面 | 作成成功 | ユーザー一覧画面 |
| ユーザー作成画面 | キャンセルボタン | ユーザー一覧画面 |

## 4. アクセス権限

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| ユーザー名 | submit | 必須・1〜50文字 | ERR_VAL_001 | ユーザー名を入力してください |
| メールアドレス | submit | 必須・メール形式・テナント内重複なし | ERR_VAL_001 | 有効なメールアドレスを入力してください |
| パスワード | submit | 必須・8文字以上 | ERR_VAL_001 | パスワードは8文字以上で入力してください |
| ロール | submit | admin / member のいずれか | ERR_VAL_001 | ロールを選択してください |

## 8. アクション

### 画面表示時

なし

### 作成ボタン押下時

1. フロント側バリデーション実行
2. `POST /api/v1/auth/users/` 呼び出し
3. 成功（201）: `/users` へ遷移し、成功トーストを表示
4. 失敗（400）: エラーメッセージを表示
