# 【API設計書】自動割り振りプレビュー生成

## 1. 概要

プロジェクトのタスクをメンバーに自動割り振りした結果をプレビューとして返す。DBへの保存は行わない。admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/auto-assign/preview/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### ボディ

なし（プロジェクトの現在状態から自動算出）

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "preview": [
    {
      "task_id": 10,
      "task_title": "フロントエンド実装",
      "assigned_to": {
        "user_id": 2,
        "username": "taro"
      }
    },
    {
      "task_id": 11,
      "task_title": "バックエンド実装",
      "assigned_to": {
        "user_id": 1,
        "username": "hanako"
      }
    }
  ]
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **データ取得**
   - 担当者未割り当てのタスク一覧を `project_id` でフィルタして SELECT
   - プロジェクトメンバー一覧を SELECT
     - DBエラー：500 `ERR_SYS_500` を返す
3. **割り振りロジック**
   - 第1層（大項目）単位で優先度を考慮
   - 優先度の高い大項目から処理
   - 大項目内のタスクをメンバーに均等に割り振り
4. **レスポンス生成**
   - 割り振り結果をプレビューとして返却（DBには保存しない）
