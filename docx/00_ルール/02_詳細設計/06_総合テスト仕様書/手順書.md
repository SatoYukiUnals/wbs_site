# 詳細設計：総合テスト仕様書

## 1. ■概要・目的

複数のプログラムや画面を組み合わせた「業務シナリオ」が、要件通りに正しく動作することを検証します。Frontend、Backend、Database、外部システムまでを統合した環境でのデータ整合性と一貫性を担保します。本プロジェクトでは、**「テスト項目一覧」** と **「データ一覧」** の2ドキュメントで構成し、業務シナリオと前提データを分離して管理します。

結合テストが画面・API単位の動作検証であるのに対し、総合テストは「受注→承認→出荷」等の業務フロー全体を通じた E2E 検証を対象とします。

**シナリオ設計の方針**：機能仕様書に記載された主要な分岐（正常フロー・承認ルート・エラー時フロー等）をシナリオとして網羅します。結合テストのような全パターン網羅ではなく、業務として意味のある代表的な分岐を各シナリオに割り当てる粒度で設計します。

1. **テスト項目一覧** ([テンプレート](./テンプレート/テスト項目一覧_template.md))：業務シナリオを項番順に実施するためのメインドキュメント
2. **データ一覧** ([テンプレート](./テンプレート/データ一覧_template.md))：テスト実施の都度データ生成をしないよう、**事前に作成しておく前提データ** を箇条書きで列挙する（期待値はテスト項目一覧側に記載）
3. **（任意）テスト項目一覧（網羅）**：シナリオが多く一覧で俯瞰したい場合、別ファイルで表形式にまとめる

## 3. ■作業手順（Issueのタスクリストとして利用）

- [ ] **業務シナリオの策定**：機能仕様書の主要な分岐（正常フロー・承認ルート・エラー時フロー等）を洗い出し、各分岐を網羅するE2Eシナリオを策定します。全パターン網羅ではなく、業務として意味のある代表的な分岐を1シナリオに割り当てる粒度で設計します。
- [ ] **データ一覧の作成**：[データ一覧_template.md](./テンプレート/データ一覧_template.md) をコピーして、テスト実施前に **事前に作成しておくユーザー・マスタ・トランザクションデータ** を箇条書きで列挙します。各ケースには `Case N` 形式のIDを振ります（期待値はテスト項目一覧側に記載）。
- [ ] **テスト項目一覧の作成**：[テスト項目一覧_template.md](./テンプレート/テスト項目一覧_template.md) をコピーして、シナリオのステップごとに「前提条件」「操作者」「テスト内容」「期待結果（画面・外部連携）」を記載します。前提条件はデータ一覧の `Case No.` を参照します。
- [ ] **環境・ロールの定義**：実行OS・ブラウザ（互換性）、各ユーザー権限別のアクセス制御をテスト対象に含めます。
- [ ] **（任意）網羅一覧の作成**：シナリオが多く、全体を俯瞰したい場合は別ファイルで表形式の一覧を作成します。

## 4. ■記載ルール

- テスト内容・期待結果に**画面仕様書・設計書への参照は記載しない**。期待結果は具体的な値・状態・HTTPステータスで記述する。
  - NG：「仕様書の定義と一致していること」
  - OK：「〇〇一覧画面に遷移し、登録したデータが1行目に表示されていること」
- **期待結果はコードでテストが書けるレベルの具体性で記述する**。「正しく表示されること」のような曖昧な表現は使わない。
- **登録・更新・削除の結果確認はGUI（詳細画面・一覧画面等）で行う**。DBを直接参照・確認しない。
  - NG：「データベースに1件登録されていること」
  - OK：「詳細画面に入力した値が表示されていること」
- **外部連携（メール・API）の確認**は以下の方法で行う。画面上で確認できる場合はGUIを優先する。
  - メール送信：受信トレイで件名・宛先・本文を目視確認する
  - API連携：連携先システムの管理画面やログで確認する
- テスト共通のドキュメント（データ一覧の `Case No.` 等）への参照はこの限りではない。

## 5. ■セルフチェック項目（作業完了後の最終確認）

- [ ] 「テスト項目一覧」と「データ一覧」の2ドキュメントが作成されている
- [ ] 機能仕様書の主要な分岐（正常フロー・承認ルート・エラー時フロー等）がシナリオとして網羅されている
- [ ] 各ステップの期待結果が具体的（画面項目、メール送信等）で、チェックボックス形式になっている
- [ ] 複数ロールを跨ぐシナリオでは、各ステップの操作者が明記されている
- [ ] 前提条件となるデータがデータ一覧に定義されており、テスト実行前に作成可能な形で記載されている
- [ ] 権限（ロール）による制御やエラーハンドリングの検証が含まれている
- [ ] 外部連携（メール・API・ファイル出力など）の副作用がテスト項目一覧の期待結果に含まれている
- [ ] テスト項目一覧の「前提条件」とデータ一覧の `Case No.` が整合している
- [ ] 期待値はテスト項目一覧の「期待結果」に記載されており、データ一覧には含めていない
- [ ] シナリオ間に実施依存関係がある場合、先行シナリオが明記されている
- [ ] ブラウザ互換性や、想定される動作環境の検証項目が含まれている
- [ ] テンプレートの `> [記]` で始まる行をすべて削除している
- [ ] AIによる自動生成内容を人間がレビューし、必要に応じて修正・補足されている
- [ ] ユーザー（角田さん）、PMが見てシナリオの網羅性や妥当性を判断できる内容になっている
