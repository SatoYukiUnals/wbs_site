# 実装：DB（モデル・マイグレーション）

## 1. ■作業手順

- [ ] **モデル・フィールドの実装**: テーブル定義書に定義されたテーブル・カラム・型・制約を `models.py` に実装します。
- [ ] **マイグレーションファイルの作成**: `makemigrations` を実行してマイグレーションファイルを生成します。
- [ ] **マイグレーションの安全性確認**: `NOT NULL` カラムの追加・カラム削除・型変更がある場合は、デフォルト値の設定・段階的マイグレーションの要否を確認します。
- [ ] **ローカル環境での適用確認**: `migrate` を実行し、エラーなく適用されること・`migrate --run-syncdb` で意図しない差分が残っていないことを確認します。

---

## 2. ■セルフチェック

- [ ] テーブル・カラム定義（型・NULL許容・デフォルト値・制約）がテーブル定義書と一致している
- [ ] インデックスがテーブル定義書の定義通りに設定されている
- [ ] `NOT NULL` カラムの追加・型変更・カラム削除がある場合、デフォルト値の設定・段階的マイグレーションの要否を確認している
- [ ] マイグレーションファイルがローカル環境で正常に apply・revert できる
- [ ] マイグレーション適用後、既存データに不整合が生じていない（既存レコードの値・リレーションが壊れていないことを画面上で確認している）
- [ ] 不要なフィールド・モデルが実装に含まれていない
