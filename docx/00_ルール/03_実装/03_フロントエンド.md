# 実装：フロントエンド（Vue.js）

コーディング規約（`<script setup>` の記述順・Template属性の記載順・命名規則・スタイル等）は [09_開発規約/01_コーディング規約_Vue.md](../../09_開発規約/01_コーディング規約_Vue.md) に従う。本ドキュメントは実装フロー・テスト・セルフチェックに絞る。

## 1. ■設計方針

### テスト

- **単体テスト**は **Vitest** で実装する。
  - ファイル配置：`frontend/ordering/src/__tests__/{対象名}.spec.ts`、`frontend_ops/src/__tests__/{対象名}.spec.ts`
  - 対象：コンポーネント・コンポーザブル・バリデーション関数・ストア等のロジック単位
- **E2Eテスト**は **Playwright** で実装する。
  - ファイル配置：`frontend/ordering/e2e/{機能名}.spec.ts`、`frontend_ops/e2e/{機能名}.spec.ts`
  - 対象：画面遷移を伴う一連のユーザー操作（ログイン〜主要機能の操作〜結果確認）

---

## 2. ■作業手順

- [ ] **APIクライアントの実装**: バックエンドのエンドポイントを呼び出す関数を実装します。
- [ ] **画面コンポーネントの実装**: 画面仕様書に定義された項目・レイアウト・操作に従い、コンポーネントを実装します。
- [ ] **入力バリデーションの実装**: 画面仕様書に定義されたバリデーションルール（必須チェック、桁数制限等）を実装します。
- [ ] **エラー・成功メッセージの表示**: APIレスポンスに応じて、エラーメッセージ仕様書に定義された文言・表示位置でメッセージを表示します。
- [ ] **単体テスト（Vitest）の実装・実行**: バリデーション関数・コンポーザブル等のロジック単位で単体テストを実装し、`npm run test:unit` で全件グリーンであることを確認します。
- [ ] **E2Eテスト（Playwright）の実装・実行**: 画面遷移を伴う主要なユーザー操作のE2Eテストを実装し、`npm run test:e2e` で全件グリーンであることを確認します。
- [ ] **ロール別表示制御の確認**: 画面仕様書に定義されたロールごとのボタン表示・非表示が正しく動作することをブラウザ上で確認します。
- [ ] **正常系・異常系の動作確認**: ブラウザ上でゴールデンパス（正常系）および主要な異常系（バリデーションエラー、権限エラー等）の動作を目視確認します。

---

## 3. ■セルフチェック

### 設計との整合

- [ ] 画面仕様書に定義された項目・レイアウト・操作と実装が一致している
- [ ] バリデーションルール（必須・桁数・形式等）が画面仕様書と一致している
- [ ] エラーメッセージ・成功メッセージの文言・表示位置がエラーメッセージ仕様書と一致している

### 動作確認

- [ ] ロール別のボタン表示・非表示が正しく動作することをブラウザ上で確認している
- [ ] 正常系（ゴールデンパス）の動作をブラウザ上で確認している
- [ ] 主要な異常系（バリデーションエラー・権限エラー等）の動作をブラウザ上で確認している

### テスト

- [ ] 単体テスト（Vitest）が実装・実行されており、すべてパスしている
- [ ] E2Eテスト（Playwright）が実装・実行されており、すべてパスしている
- [ ] 既存のテストが全件パスしており、デグレードがないことを確認している

### コード品質

- [ ] [コーディング規約（Vue.js）](../../09_開発規約/01_コーディング規約_Vue.md) に従って実装している（`<script setup>` の記述順・Template属性の記載順・命名規則 等）
- [ ] `console.log` 等のデバッグコードが残っていない
- [ ] ユーザー入力値を直接HTMLに埋め込んでいない（XSS対策）
