# 【API設計書】クォーター詳細取得

## 1. 概要

指定クォーターの詳細情報（タイトル・期間・進捗率）を返す。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/{project_id}/quarters/{quarter_id}/`
- **認証:** JWT必須（プロジェクトメンバー以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

#### `quarter_id`（クォーターID）

- **型:** `integer` / **必須:** ○

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "title": "Q1",
  "start_date": "2026-04-01",
  "end_date": "2026-06-30",
  "progress": 50,
  "project_id": 1
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・メンバー確認**
   - `ProjectMember` でメンバーシップを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 非メンバー：403 `ERR_PERM_001` を返す
2. **データ取得**
   - `Quarter` を `quarter_id` AND `project_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
3. **レスポンス生成**
   - クォーター詳細を返却
