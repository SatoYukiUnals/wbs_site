# 【API設計書】プロジェクトメンバー削除

## 1. 概要

指定プロジェクトからメンバーを除外する。プロジェクト内の admin 以上のみ実行可能。自分自身の削除は不可。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/projects/{project_id}/members/{user_id}/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

#### `user_id`（削除対象ユーザーID）

- **型:** `integer` / **必須:** ○

## 4. レスポンス

### 成功 (204 No Content)

レスポンスボディなし

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 自分自身の削除 | 403 | `ERR_PERM_003` | 自分自身は削除できません | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **自己削除チェック**
   - `user_id` が JWT の `user_id` と一致しないことを確認
     - 一致：403 `ERR_PERM_003` を返す
3. **対象メンバー確認**
   - `ProjectMember` で `project_id` AND `user_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
4. **DB削除**
   - `ProjectMember` レコードを DELETE
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 204 No Content を返却
