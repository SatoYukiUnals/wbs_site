# 【API設計書】プロジェクト内レビュー一覧取得

## 1. 概要

指定プロジェクト内の全レビュー一覧（レビュー対象タスク・ステータス・レビュアー）を返す。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/{project_id}/reviews/`
- **認証:** JWT必須（プロジェクトメンバー以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

### クエリパラメータ

#### `status`（レビューステータスフィルタ）

- **型:** `string` / **必須:** -
- `pending` / `approved` / `rejected` / `確認待ち` / `完了` のいずれか

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "reviews": [
    {
      "id": 1,
      "task_id": 10,
      "task_title": "フロントエンド実装",
      "status": "rejected",
      "reviewer_id": 2,
      "reviewer_name": "taro",
      "created_at": "2026-04-15T10:00:00Z"
    }
  ]
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・メンバー確認**
   - `ProjectMember` でメンバーシップを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 非メンバー：403 `ERR_PERM_001` を返す
2. **データ取得**
   - `Review` を `Task.project_id = project_id` + statusフィルタ（指定時）で SELECT
   - `Task`・`User` を JOIN してタスク名・レビュアー名を取得
     - DBエラー：500 `ERR_SYS_500` を返す
3. **レスポンス生成**
   - レビュー一覧を返却
