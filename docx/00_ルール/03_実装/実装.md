# 実装：手順書

## 1. ■概要・目的

詳細設計書（API設計、画面仕様書、機能仕様書等）をもとに、バックエンド（Django）およびフロントエンド（Vue.js）の実装を行います。設計書との乖離を防ぎ、品質を担保した状態でコードを完成させることを目的とします。

各工程の詳細な手順・セルフチェックは以下の工程別ドキュメントを参照してください。

## 3. ■事前準備（全工程共通）

- [ ] 詳細設計（API設計書、機能仕様書、画面仕様書、単体テスト仕様書）が完成・レビュー済みであること
- [ ] ローカル開発環境が起動できる状態であること（DB・サーバー含む）
- [ ] 実装対象のIssueが起票・アサインされていること
- [ ] 作業ブランチが `main` から `feature/XXXX-説明`（Issue番号 + 英語）の形式で切られていること

---

## 4. ■共通セルフチェック（PR作成前）

- [ ] 設計書と実装に差異がある場合、設計書が更新されている
- [ ] 不要なデバッグコード（`console.log`、`print`、コメントアウトされたコード等）が含まれていない
- [ ] PRの説明に変更概要・動作確認方法・スクリーンショット（画面変更がある場合）が記載されている
- [ ] AIによる自動生成コードを人間がレビューし、意図しない処理が含まれていないことを確認している

---
