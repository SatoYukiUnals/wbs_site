# 【API設計書】ユーザー作成

## 1. 概要

同テナント内に新しいユーザーを作成する。マスターユーザーのみ実行可能。作成時にロール（admin / member）を指定する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/users/`
- **認証:** JWT必須（masterのみ）

## 3. リクエスト

### ボディ

#### `username`（ユーザー名）

- **型:** `string` / **必須:** ○
- 1〜150文字

#### `email`（メールアドレス）

- **型:** `string` / **必須:** ○
- メール形式。全テナント横断で一意

#### `password`（パスワード）

- **型:** `string` / **必須:** ○
- 8文字以上・英数字混在

#### `role`（ロール）

- **型:** `string` / **必須:** ○
- `admin` または `member` のいずれか（`master` は指定不可）

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "id": 3,
  "username": "jiro",
  "email": "jiro@example.com",
  "role": "member",
  "tenant_id": 1,
  "created_at": "2026-04-17T00:00:00Z"
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| メールアドレス重複 | 400 | `ERR_VAL_003` | このメールアドレスは既に使用されています | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限チェック**
   - `role` が `master` であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **バリデーション**
   - 全必須項目の存在チェック
   - `email` 形式チェック
   - `role` が `admin` または `member` であることを確認
   - `password` 強度チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **メールアドレス重複チェック**
   - `User` テーブル全体で `email` 未使用を確認
     - 重複：400 `ERR_VAL_003` を返す
4. **DB操作**
   - `User` レコードを INSERT（`tenant_id` = JWTの `tenant_id`・パスワードはハッシュ化）
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 作成したユーザー情報を返却
