# 【API設計書】メンバー追加API

## 1. 概要

ユーザーをプロジェクトメンバーに追加し、ロールを設定する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/{project_id}/members/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクト内 admin 以上

## 3. リクエスト

### ボディ（JSON）

#### `user_id`

- **型:** `UUID` / **必須:** ✅
- 同テナント内のユーザーIDであること

#### `role`

- **型:** `string` / **必須:** ✅
- `admin` または `member`

```json
{
  "user_id": "uuid",
  "role": "member"
}
```

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "user_id": "uuid",
  "username": "田中花子",
  "role": "member"
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 対象ユーザーがテナント外 | 400 | 対象ユーザーは同テナント内のユーザーでなければなりません |
| 既にメンバー | 409 | このユーザーは既にメンバーです |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - プロジェクトメンバーかつ `role=admin` 以上
2. **バリデーション**
   - `user_id` が同テナント内であることを確認。テナント外：400 を返す
   - 既に `project_member` レコードが存在：409 を返す
3. **DB操作**
   - `project_member` レコードを INSERT
4. **レスポンス**
   - 追加したメンバー情報を 201 で返す
