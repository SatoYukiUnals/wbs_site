# 【API設計書】プロジェクト編集API

## 1. 概要

プロジェクトの名前・説明・開始日・終了日を更新する。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/projects/{project_id}/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクト内 admin 以上

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `project_id` | UUID | ✅ | 対象プロジェクトID |

### ボディ（JSON）

`projects_POST.md` のリクエスト仕様と同一。

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "project_id": "uuid",
  "name": "ECサイトリニューアル（更新）",
  "description": "更新された説明",
  "start_date": "2026-04-01",
  "end_date": "2026-09-30",
  "progress": 40
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| プロジェクト存在しない | 404 | プロジェクトが見つかりません |
| `end_date` ≤ `start_date` | 400 | 終了日は開始日より後に設定してください |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - プロジェクトメンバーかつ `role=admin` 以上。それ以外：403 を返す
2. **バリデーション**
   - 終了日 > 開始日チェック
3. **DB更新**
   - `project` テーブルを UPDATE
4. **レスポンス**
   - 更新後のプロジェクト情報を返す
