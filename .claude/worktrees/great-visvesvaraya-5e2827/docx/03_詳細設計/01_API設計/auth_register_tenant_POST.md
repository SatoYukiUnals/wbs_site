# 【API設計書】テナント作成・マスターユーザー登録

## 1. 概要

新規テナントを作成し、そのテナントのマスターユーザーを同時に登録する。認証不要のパブリックエンドポイント。テナント名・ユーザー情報のバリデーション後、Tenant・Userレコードを一括作成してJWTトークンを返す。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/register/tenant/`
- **認証:** 不要

## 3. リクエスト

### ボディ

#### `tenant_name`（テナント名）

- **型:** `string` / **必須:** ○
- 1〜100文字

#### `username`（ユーザー名）

- **型:** `string` / **必須:** ○
- 1〜150文字

#### `email`（メールアドレス）

- **型:** `string` / **必須:** ○
- メール形式。全テナント横断で一意

#### `password`（パスワード）

- **型:** `string` / **必須:** ○
- 8文字以上・英数字混在

#### `password_confirm`（パスワード確認）

- **型:** `string` / **必須:** ○
- `password` と一致すること

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "tenant": {
    "id": 1,
    "name": "株式会社サンプル"
  },
  "user": {
    "id": 1,
    "username": "hanako",
    "email": "hanako@example.com",
    "role": "master"
  },
  "access_token": "eyJ...",
  "refresh_token": "eyJ..."
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | 不備のある項目名を含む |
| パスワード不一致 | 400 | `ERR_VAL_002` | パスワードが一致しません | |
| メールアドレス重複 | 400 | `ERR_VAL_003` | このメールアドレスは既に使用されています | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **バリデーション**
   - 全必須項目の存在チェック
   - `email` 形式チェック
   - `password` 強度チェック（8文字以上・英数字混在）
   - `password` と `password_confirm` の一致チェック
     - エラー時：400 `ERR_VAL_001` / `ERR_VAL_002` を返す
2. **メールアドレス重複チェック**
   - `User` テーブル全体で `email` が未使用であることを確認
     - 重複時：400 `ERR_VAL_003` を返す
3. **DB操作（トランザクション）**
   - `Tenant` レコードを INSERT（`name` = `tenant_name`）
   - `User` レコードを INSERT（`role` = `master`・`tenant_id` = 作成したTenantのID・パスワードはハッシュ化）
     - DBエラー：500 `ERR_SYS_500` を返す
4. **JWT発行**
   - ペイロードに `user_id`・`tenant_id`・`role` を含める
   - アクセストークン（15分）・リフレッシュトークン（7日）を発行
   - リフレッシュトークンを Redis に保存
5. **レスポンス生成**
   - tenant・user情報・トークンペアを返却
