# WBS管理ソフト 要件定義書 v1.9

**技術スタック：** Django / Vue.js / PostgreSQL  
**作成日：** 2026年4月12日  
**バージョン：** 1.9

---

## 1. システム概要

テナント単位で完全独立したデータ管理を行う、テナント単位で完全独立したデータ管理を行う、小規模チーム（2〜10人）向けの複数プロジェクト横断WBS管理Webアプリ。タスクの階層管理（最大5層）・クォーター管理・ガントチャート可視化・プロダクトロードマップ・進捗管理・レビュー管理・タスク自動割り振り・テンプレート管理・Excel出力・定期報告書自動作成・PDF出力を提供する。

---

## 2. 利用者・権限設計

| ロール | 権限内容 |
|--------|---------|
| 管理者（Admin） | プロジェクト作成・編集・削除、メンバー管理、全タスク編集、全タスクレビュー、自動割り振り実行、テンプレート共有設定、Excel出力、報告書の作成・出力 |
| メンバー（Member） | 担当タスクの進捗更新・ステータス変更、自分以外のタスクのレビュー、テンプレートの作成・利用、Excel出力、報告書の閲覧 |

---

## 3. 機能一覧

### 3-1. 認証・ユーザー管理
- ユーザー登録・ログイン（JWT認証）
- ログアウト・トークンリフレッシュ
- プロフィール管理（氏名・メールアドレス変更）

### 3-2. プロジェクト管理
- プロジェクトの作成・編集・削除・一覧表示
- メンバーの招待・ロール設定・削除
- プロジェクト全体の進捗率の自動集計・表示

### 3-3. クォーター管理
- プロジェクト内を Q1 / Q2 / Q3 / Q4 などのクォーター単位で区切る
- クォーターに開始日・終了日・タイトルを設定
- タスクをクォーターに紐付け可能
- ガントチャート上にクォーターの区切りを表示
- クォーターごとの進捗率を集計・表示

### 3-4. WBS（タスク階層管理）

最大5階層のタスク構造：

| 階層 | 例 |
|------|----|
| 第1層（大項目） | 1. 要件定義 |
| 第2層（中項目） | 1.1 ヒアリング |
| 第3層（小項目） | 1.1.1 ユーザーインタビュー |
| 第4層（詳細） | 1.1.1.1 質問票作成 |
| 第5層（作業単位） | 1.1.1.1.1 ドラフト作成 |

- 6層目以降の追加をバックエンド・フロントエンド双方でバリデーション制御
- タスクの作成・編集・削除
- タスク属性：タイトル、説明、開始日、終了日、工数（時間）、ステータス、進捗率（0〜100%）、優先度、担当者（複数可）、クォーター紐付け
- ドラッグ＆ドロップによる並び替え

**ステータス定義**

| ステータス | 説明 |
|-----------|------|
| 未着手 | タスクがまだ開始されていない状態 |
| 進行中 | 現在作業中の状態 |
| レビュー待ち | 作業完了後、承認・確認待ちの状態 |
| 完了 | タスクが完了した状態 |
| 保留 | 何らかの理由で一時停止している状態 |

**優先度定義**

| 優先度 | 説明 |
|--------|------|
| 高 | 最優先で対応が必要 |
| 中 | 通常優先度 |
| 低 | 余裕があれば対応 |

### 3-5. タスク自動割り振り
- 管理者が実行ボタンで自動割り振りを起動
- 割り振りロジック：

```
Step 1: 大項目（第1層）単位で優先度を設定（高・中・低）
Step 2: 優先度の高い大項目から順に処理
Step 3: 大項目内のタスクを各メンバーに均等に割り振り
Step 4: 割り振り結果をプレビューで確認 → 管理者が承認して確定
```

- 自動割り振り後も手動での変更は可能
- 割り振り履歴を記録・参照可能

### 3-6. ガントチャート
- タスクのスケジュールを時系列で可視化（閲覧専用）
- 進捗率をバーで表示
- クォーターの区切りを表示
- 月・週単位での表示切り替え

### 3-7. プロダクトロードマップ
- プロジェクト単位でクォーターを軸にしたロードマップを表示（閲覧専用）
- クォーターごとに機能・テーマを配置して可視化
- ロードマップアイテムにタイトル・説明・クォーター・ステータスを設定
- WBSのタスクとは独立した、計画・方向性の可視化用途

### 3-8. 進捗・ステータス管理
- ダッシュボードで全プロジェクトの進捗を一覧表示
- プロジェクト・クォーター・タスクごとの進捗率を自動集計
- 担当者・ステータス・クォーターによるフィルタリング

### 3-9. レビュー管理

レビューフロー：
```
進行中 → レビュー待ち → 承認（完了）
                     → 差し戻し（進行中に戻る）
```

- タスク担当者本人以外のメンバー全員がレビュー可能

| 機能 | 内容 |
|------|------|
| 承認 | タスクを「完了」に移行 |
| 差し戻し | タスクを「進行中」に戻す |
| コメント | 承認・差し戻し時に理由・メモを記入 |
| 履歴管理 | 修正依頼・承認・差し戻しの履歴を時系列で記録・閲覧 |

### 3-10. 報告書管理
- 任意期間・クォーター単位の進捗レポートを作成・保存・閲覧
- PDF形式での出力
- レポート履歴の一覧管理
- 一定期間ごとの報告書自動作成（定期レポート）

**定期レポート設定**

| 設定項目 | 内容 |
|---------|------|
| 作成頻度 | 週次 / 隔週 / 月次 / クォーターから選択 |
| 対象プロジェクト | 1つまたは複数のプロジェクトを指定 |
| 自動作成タイミング | 期間終了時に自動生成 |
| 通知 | 作成完了時にメンバーへ通知 |

### 3-11. WBS Excel出力 ★NEW
- WBSの内容をExcel（.xlsx）形式でエクスポート
- 出力内容：タスク階層・タイトル・担当者・開始日・終了日・工数・ステータス・進捗率
- 階層をインデントで表現
- プロジェクト単位 / クォーター単位での出力が可能

### 3-12. テンプレート管理

#### ① WBSテンプレート（直下1層一括生成）
- 選択したタスクの直下1層分のみを一括生成
- 改行区切りのテキストで子タスクを定義

```
フロントエンド実装
バックエンド実装
テスト確認
レビュー依頼
```

- テンプレートを適用すると、選択タスクの子として上記を一括追加
- テンプレートはプロジェクト作成時 or 後から適用可能

#### ② ユーザー定義テンプレート
- ユーザーが独自テンプレートを作成・保存・編集・削除
- プロジェクト横断で再利用可能
- 管理者が作成したテンプレートはチーム全体で共有可能

#### ③ タスクテンプレート（Issue形式）
- テンプレートを選択してタスクを1件作成（GitHubのIssueを切る感覚）
- 対応内容は改行区切りのテキストで定義

```
仕様確認
実装
動作確認
レビュー依頼
```

- 改行ごとに1項目として認識
- タスクの説明フィールドにそのまま格納

## 4. データモデル設計

| テーブル名 | 主なカラム | 説明 |
|-----------|-----------|------|
| Tenant | id, name, created_at | テナント情報 |
| User | id, tenant_id, username, email, password, role（master/admin/member） | ユーザー情報（テナントに紐付け） |

| Project | id, name, description, start_date, end_date, created_by | プロジェクト情報 |
| ProjectMember | id, project_id, user_id, role | プロジェクト参加者とロール |
| Quarter | id, project_id, title, start_date, end_date | クォーター情報 |
| RoadmapItem | id, project_id, quarter_id, title, description, status | ロードマップアイテム |
| Task | id, project_id, quarter_id, parent_task_id, title, description, order, start_date, end_date, estimated_hours, status, progress, priority | タスク（自己参照で階層表現、最大5層） |
| TaskAssignee | id, task_id, user_id | タスク担当者 |
| TaskStatusHistory | id, task_id, status, changed_at, changed_by | ステータス変更履歴（実績期間の算出に使用） |
| AutoAssignLog | id, project_id, executed_by, executed_at, result（JSON） | 自動割り振り履歴 |
| Review | id, task_id, reviewer_id, status, comment, created_at | レビュー結果 |
| ReviewHistory | id, task_id, action, comment, actor_id, created_at | レビュー履歴 |
| WBSTemplate | id, title, content（text）, created_by, is_shared | WBS子タスク一括生成テンプレート |
| TaskTemplate | id, title, content（text）, created_by, is_shared | タスク（Issue）テンプレート |
| Report | id, project_id, title, period_start, period_end, content, created_by, created_at, is_auto | 報告書（is_auto=定期自動作成フラグ） |
| ReportSchedule | id, project_id, frequency（weekly/biweekly/monthly/quarterly）, next_run, created_by | 定期レポートスケジュール |

## 5. 技術スタック

| レイヤー | 技術・ライブラリ |
|---------|----------------|

## 7. 制約・非機能要件

| 項目 | 内容 |
|------|------|
| 利用者規模 | 最大10名程度の小規模チーム |
| WBS階層上限 | 最大5層（バックエンド・フロントエンド双方でバリデーション） |
| WBSテンプレート生成 | 直下1層のみ一括生成 |
| タスクテンプレート形式 | 改行区切りのテキスト（対応内容のみ） |
| ガントチャート編集 | 閲覧のみ（編集不可） |
| 報告書出力形式 | PDFのみ |
| WBS出力形式 | Excelのみ（.xlsx） |
| 定期レポート頻度 | 週次 / 隔週 / 月次 / クォーター |
| 対応ブラウザ | Chrome / Firefox / Edge 最新版 |
| 認証方式 | JWT（アクセストークン＋リフレッシュトークン） |
| 自動割り振り | 管理者のみ実行可、実行前にプレビュー確認あり |
