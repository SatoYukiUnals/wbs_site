# 【単体テスト：共通テスト項目】

## 概要

CRUD操作全般に共通するテストパターンをここに集約する。  
各個別テスト仕様書では「→ 共通A-1参照」のように参照し、重複記載を避ける。

---

## 共通A：全操作共通（認証・テナント分離）

| No | 区分 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| A-1 | 未認証 | Authorizationヘッダーなしでリクエストする | `401 Unauthorized` / `ERR_AUTH_002` |
| A-2 | 無効トークン | 改ざん済み / 期限切れのJWTでリクエストする | `401 Unauthorized` / `ERR_AUTH_002` |
| A-3 | テナント分離 | 別テナントが所有するリソースのIDを指定してリクエストする | `404 Not Found`（存在しないものとして扱う） |

---

## 共通B：ID指定操作共通（詳細・変更・削除）

| No | 区分 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| B-1 | 対象未存在 | 存在しないIDを指定してリクエストする | `404 Not Found` / `ERR_DAT_404` |
| B-2 | ID不正形式 | 数値でないIDを指定してリクエストする（例: `abc`） | `400 Bad Request` / `ERR_VAL_001` |

---

## 共通C：登録・変更共通（権限・バリデーション）

| No | 区分 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| C-1 | 権限不足 | 必要ロール未満のユーザーで作成・変更リクエストをする | `403 Forbidden` / `ERR_PERM_001` |
| C-2 | 必須項目未入力 | 必須フィールドを空にしてリクエストする | `400 Bad Request` / `ERR_VAL_001`、対象フィールドのエラーメッセージあり |
| C-3 | 文字数超過 | 最大文字数+1の文字列を指定してリクエストする | `400 Bad Request` / `ERR_VAL_001` |
| C-4 | 最大文字数境界値 | 最大文字数ちょうどの文字列を指定してリクエストする | 正常処理される |

---

## 共通D：一覧取得共通

| No | 区分 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| D-1 | 0件 | 対象データが存在しない状態で一覧を取得する | `200 OK`、レスポンスの該当配列が空（`[]`） |
| D-2 | テナント分離 | 別テナントのデータが複数存在する状態で一覧を取得する | 自テナントのデータのみ返却される（件数・内容で確認） |

---

## 共通E：削除共通

| No | 区分 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| E-1 | 削除後の再取得 | 削除後に同IDで詳細取得リクエストをする | `404 Not Found` / `ERR_DAT_404` |
| E-2 | 権限不足 | 必要ロール未満のユーザーで削除リクエストをする | `403 Forbidden` / `ERR_PERM_001` |

---

## 適用ガイド

各個別テスト仕様書では、以下のように参照する。

```
→ 共通A-1、A-2、A-3 参照
→ 共通B-1、B-2 参照
→ 共通C-1、C-2、C-3、C-4 参照
```

個別仕様書固有のテストケース（ビジネスロジック、階層構造、固有バリデーション等）のみ個別ファイルに記載する。
