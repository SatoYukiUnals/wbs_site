# 【API設計書】プロフィール更新

## 1. 概要

ログイン中ユーザー自身のプロフィール（ユーザー名・メールアドレス・パスワード）を更新する。パスワード変更時は現在のパスワードによる確認を必須とする。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/auth/profile/`
- **認証:** JWT必須（全ユーザー）

## 3. リクエスト

### ボディ

#### `username`（ユーザー名）

- **型:** `string` / **必須:** -
- 1〜150文字

#### `email`（メールアドレス）

- **型:** `string` / **必須:** -
- メール形式。全テナント横断で一意

#### `password`（新しいパスワード）

- **型:** `string` / **必須:** -
- 8文字以上・英数字混在。指定時は `current_password` も必須

#### `current_password`（現在のパスワード）

- **型:** `string` / **必須:** △（`password` 指定時のみ必須）

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "username": "hanako_updated",
  "email": "hanako_new@example.com",
  "role": "admin",
  "tenant_id": 1
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 現在のパスワード不一致 | 400 | `ERR_VAL_005` | 現在のパスワードが正しくありません | |
| メールアドレス重複 | 400 | `ERR_VAL_003` | このメールアドレスは既に使用されています | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `user_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **バリデーション**
   - 変更対象フィールドの形式チェック
   - `password` 指定時：`current_password` の存在チェック
     - エラー時：400 `ERR_VAL_001` を返す
3. **現在のパスワード照合**（`password` 指定時のみ）
   - `current_password` をハッシュ照合
     - 不一致：400 `ERR_VAL_005` を返す
4. **メールアドレス重複チェック**（`email` 変更時のみ）
   - `User` テーブル全体で `email` 未使用を確認
     - 重複：400 `ERR_VAL_003` を返す
5. **DB更新**
   - 指定フィールドのみ UPDATE（パスワードはハッシュ化して保存）
     - DBエラー：500 `ERR_SYS_500` を返す
6. **レスポンス生成**
   - 更新後のユーザー情報を返却
