# 【総合テスト：テスト項目一覧】（対象：報告書作成・PDF出力 / WBS Excel出力フロー）

## 1. シナリオ概要

プロジェクトの進捗データをもとに報告書を自動生成→手動編集→PDF出力するフローと、WBS をExcelエクスポートするフローを E2E で検証する。

---

## 2. 実施環境

- **OS**：Windows 11 / macOS
- **ブラウザ**：Google Chrome（最新版）
- **先行シナリオ**：02_タスク進捗管理フロー（タスクのステータス・進捗データが揃った状態）または Case 30〜 の事前データ使用

---

## 3. テスト項目一覧

### No.1：報告書の自動生成（クォーター指定）

- **前提条件：** Case 30（プロジェクト）、Case 31（Q1）、Case 32〜34（タスク：完了・進行中・遅延各1件）
- **操作者：** adminユーザー
- **テスト内容：**
  1. 報告書作成画面を開く
  2. 対象クォーターに「Q1」を選択する
  3. 「自動生成」ボタンを押下する
- **期待結果：**
  - [ ] 画面：「プロジェクト概要」セクションにプロジェクト名が表示されること
  - [ ] 画面：「タスク一覧」セクションにCase 32〜34 のタスク名が表示されること
  - [ ] 画面：「遅延タスク」セクションにCase 34（遅延タスク）が表示されること
  - [ ] 画面：「PDF出力」ボタンが活性状態になること
- **事後状態：** 各セクションに自動生成内容が表示されている状態

### No.2：報告書の手動編集

- **前提条件：** No.1 完了後
- **操作者：** adminユーザー
- **テスト内容：**
  1. 「プロジェクト概要」セクションのテキストを「手動追記テキスト：追加コメントあり」に編集する
- **期待結果：**
  - [ ] 画面：テキストエリアに編集内容が反映されていること
- **事後状態：** 手動編集済み状態

### No.3：PDF出力

- **前提条件：** No.2 完了後
- **操作者：** adminユーザー
- **テスト内容：**
  1. 「PDF出力」ボタンを押下する
- **期待結果：**
  - [ ] 画面：`{プロジェクト名}_報告書_{出力日YYYYMMDD}.pdf` のダウンロードが開始されること
  - [ ] ダウンロードしたPDFが正常に開けること（破損していないこと）
  - [ ] PDFに「手動追記テキスト：追加コメントあり」が含まれていること

### No.4：WBS Excel出力（プロジェクト全体）

- **前提条件：** Case 30（プロジェクト）、Case 32〜34（タスク）
- **操作者：** adminユーザー
- **テスト内容：**
  1. Excel出力設定画面で出力単位「プロジェクト全体」を選択する
  2. 「Excel出力」ボタンを押下する
- **期待結果：**
  - [ ] 画面：`{プロジェクト名}_WBS_{出力日YYYYMMDD}.xlsx` のダウンロードが開始されること
  - [ ] ダウンロードしたExcelが正常に開けること
  - [ ] 「タスク一覧」シートにCase 32〜34 が含まれること
  - [ ] 「ガントチャート」シートで Case 33（actual未入力）の実績行がStatusHistoryの日付で塗られていること

### No.5：WBS Excel出力（クォーター指定）

- **前提条件：** No.4 の環境
- **操作者：** adminユーザー
- **テスト内容：**
  1. 出力単位を「クォーター指定」に変更しQ1を選択して「Excel出力」を押下する
- **期待結果：**
  - [ ] 画面：`{プロジェクト名}_WBS_{出力日}_Q1.xlsx` のダウンロードが開始されること
  - [ ] タスク一覧シートに Q1 タスクのみが含まれること

---

## 4. 備考

- テストデータ詳細は `データ一覧.md` を参照
- No.1〜3 は順番通りに実施すること（PDF出力はNo.1〜2 完了後）
- No.4・No.5 は独立して実施可能
