# 【API設計書】プロジェクト詳細API

## 1. 概要

指定プロジェクトの詳細情報（名前・説明・期間・進捗率）を取得する。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/{project_id}/`
- **認証:** 必要（Bearer トークン）
- **権限:** プロジェクトメンバー以上

## 3. リクエスト

### パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| `project_id` | UUID | ✅ | 対象プロジェクトID |

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "project_id": "uuid",
  "name": "ECサイトリニューアル",
  "description": "フルリニューアルプロジェクト",
  "start_date": "2026-04-01",
  "end_date": "2026-09-30",
  "progress": 40,
  "created_by": "uuid"
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 存在しない・テナント外・論理削除済み | 404 | プロジェクトが見つかりません |
| メンバーでない | 403 | このプロジェクトへのアクセス権がありません |

## 5. 内部処理 / ロジック

1. **テナント・メンバー確認**
   - `project` テーブルから `project_id` かつ `tenant_id` で検索・`deleted_at IS NULL`
   - `project_member` テーブルでログインユーザーがメンバーか確認
   - 存在しない：404 / メンバーでない：403 を返す
2. **レスポンス**
   - プロジェクト詳細を返す
