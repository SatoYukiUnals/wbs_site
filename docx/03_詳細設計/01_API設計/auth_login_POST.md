# 【API設計書】ログイン

## 1. 概要

メールアドレスとパスワードでユーザーを認証し、JWTトークンペア（アクセストークン・リフレッシュトークン）を返す。認証不要のパブリックエンドポイント。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/login/`
- **認証:** 不要

## 3. リクエスト

### ボディ

#### `email`（メールアドレス）

- **型:** `string` / **必須:** ○
- メール形式

#### `password`（パスワード）

- **型:** `string` / **必須:** ○

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "user": {
    "id": 1,
    "username": "hanako",
    "email": "hanako@example.com",
    "role": "admin",
    "tenant_id": 1
  },
  "access_token": "eyJ...",
  "refresh_token": "eyJ..."
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 認証失敗 | 401 | `ERR_AUTH_001` | メールアドレスまたはパスワードが正しくありません | ユーザー未存在・PW不一致を区別しない |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **バリデーション**
   - `email`・`password` の存在チェック
     - エラー時：400 `ERR_VAL_001` を返す
2. **ユーザー照合**
   - `email` で `User` レコードを取得
   - パスワードをハッシュ照合
     - ユーザー未存在 or パスワード不一致：401 `ERR_AUTH_001` を返す
3. **JWT発行**
   - ペイロードに `user_id`・`tenant_id`・`role` を含める
   - アクセストークン（15分）・リフレッシュトークン（7日）を発行
   - リフレッシュトークンを Redis に保存
4. **レスポンス生成**
   - user情報・トークンペアを返却
