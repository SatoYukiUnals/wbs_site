# 【API設計書】全プロジェクトダッシュボード取得

## 1. 概要

ログイン中ユーザーのテナント内にある全プロジェクトの進捗サマリー（進捗率・ステータス別タスク数・遅延タスク数）を返す。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/dashboard/`
- **認証:** JWT必須（全ユーザー）

## 3. リクエスト

### クエリパラメータ

なし

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "projects": [
    {
      "id": 1,
      "name": "新規システム開発",
      "progress": 35,
      "task_counts": {
        "未着手": 10,
        "進行中": 8,
        "レビュー待ち": 2,
        "完了": 15,
        "保留": 1
      },
      "delayed_count": 3
    }
  ],
  "total": {
    "project_count": 2,
    "delayed_count": 5
  }
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `tenant_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **データ取得**
   - `Project` テーブルを `tenant_id` でフィルタして全件 SELECT
   - 各プロジェクトの `Task` を集計（ステータス別件数）
   - 遅延タスク：`end_date < 今日` AND `status != 完了` の件数を集計
     - DBエラー：500 `ERR_SYS_500` を返す
3. **レスポンス生成**
   - 全プロジェクトのサマリーを返却
