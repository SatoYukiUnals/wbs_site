# 【API設計書】レビューステータス更新・コメント編集

## 1. 概要

レビューのステータス変更またはコメント本文の編集を行う。対応者（タスク担当者）または指摘者のみ操作可能。レビューが「完了」の場合は操作不可。コメント編集時は先頭に「ユーザー名 日時」を自動付与する。

## 2. エンドポイント

- **Method:** `PATCH`
- **Path:** `/api/v1/projects/{project_id}/tasks/{task_id}/reviews/{review_id}/`
- **認証:** JWT必須（対応者 or 指摘者）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

#### `task_id`（タスクID）

- **型:** `integer` / **必須:** ○

#### `review_id`（レビューID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `status`（レビューステータス）

- **型:** `string` / **必須:** -
- `確認待ち` / `完了` / `rejected` のいずれか

#### `comment_id`（編集対象コメントID）

- **型:** `integer` / **必須:** -
- `comment_body` 指定時は必須

#### `comment_body`（コメント本文）

- **型:** `string` / **必須:** -
- `comment_id` 指定時は必須

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "review": {
    "id": 1,
    "status": "確認待ち"
  }
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | 対応者・指摘者以外 |
| 完了後の操作 | 403 | `ERR_PERM_005` | 完了したレビューは変更できません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `user_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **対象レビュー確認**
   - `Review` を `review_id` AND `task_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
3. **完了チェック**
   - `Review.status` が `完了` でないことを確認
     - 完了：403 `ERR_PERM_005` を返す
4. **権限チェック**
   - リクエスト元が対応者（`TaskAssignee`）または指摘者（`ReviewComment.author_id` の一人）であることを確認
     - 権限外：403 `ERR_PERM_001` を返す
5. **DB操作（トランザクション）**
   - `status` 指定時：`Review.status` を UPDATE → `ReviewHistory` に INSERT
   - `comment_body` 指定時：`ReviewComment.body` を `"{ユーザー名} {日時}\n{新本文}\n{旧本文}"` の形式で UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
6. **レスポンス生成**
   - 更新後のレビュー情報を返却
