# 詳細設計：IF仕様書

## 1. ■概要・目的

CSV・Excel・PDFなどの形式でデータを出力するインターフェイス機能の詳細仕様を定義します。出力項目の抽出ロジック、書式、繰り返し条件等を明確にすることで、要件通りのデータ出力機能を実現可能にします。

## 2. ■事前準備

- [ ] 規定のテンプレート：[仕様書_template.md](./仕様書_template.md)
- [ ] 規定のサンプル：[仕様書_sample.md](./仕様書_sample.md)
- [ ] 項目説明テンプレート：[項目説明_template.md](./項目説明_template.md)（エンドユーザー向けマニュアル用）
- [ ] 項目説明サンプル：[項目説明_sample.md](./項目説明_sample.md)
- [ ] 出力する予定のサンプルファイル（CSV・Excel・PDFなど）を作成する

本仕様書は2つのドキュメントで構成する：

1. **IF仕様書** ([仕様書_template.md](./仕様書_template.md) / [仕様書_sample.md](./仕様書_sample.md))：実装者向けの技術仕様
2. **項目説明** ([項目説明_template.md](./項目説明_template.md) / [項目説明_sample.md](./項目説明_sample.md))：エンドユーザー・PM向けのマニュアル

## 3. ■作業手順（Issueのタスクリストとして利用）

- [ ] **テンプレートの利用**：[仕様書_template.md](./仕様書_template.md) をコピーして作成を開始します。ファイル名は `IFID_IF名.md` 形式とします。
- [ ] **基本情報の定義**：IFID・IF名・出力形式・文字コード・実行タイミングを記載します。
- [ ] **目的・対象データの記述**：IFの目的と、抽出対象のデータ範囲（期間・ステータスなど）を記載します。
- [ ] **項目一覧の定義**：出力する項目の物理名・論理名・型・繰り返しを定義します。特記事項は `▶ 備考` セクションに記載します。
- [ ] **書式の定義（Excel / PDF のみ）**：フォント・文字サイズ・太字・文字色・背景色・セル幅などを記載します。CSV出力の場合はセクションごと削除します。
- [ ] **変換・抽出ロジックの記述**：項目ごとに抽出方法・変換処理・JOIN条件を記載します。
- [ ] **参照テーブルの記述**：参照元テーブル・カラム・データ型を記載します。テーブル単位で `### ■ テーブル名` に分けます。
- [ ] **全体備考の記述**：0件時の挙動、最大件数の制限、ファイル名の命名規則などを記載します。
- [ ] **項目説明の作成**：[項目説明_template.md](./項目説明_template.md) をコピーして、エンドユーザー向けのマニュアル形式で出力項目の説明を記載します。
  - ファイル名：`IFID_IF名_項目説明.md`
  - 出力ファイルの構成単位でセクションを区切る（①、②…）
  - 必ず出力される項目には `❗` をつける
  - 技術用語を避け、出力内容を理解するための説明として記述する
  - 補足は `💡`、注意は `⚠️` のマーカーを使い、blockquote で強調する

## 4. ■セルフチェック項目（作業完了後の最終確認）

- [ ] 出力ファイルのサンプル（イメージ）が用意されている
- [ ] すべての出力項目に対して、参照先テーブル・カラムや変換ロジックが定義されている
- [ ] 繰り返し・ソート順・抽出条件が明確になっている
- [ ] Excel / PDF 出力の場合、書式（フォント・文字サイズ・セル幅など）が定義されている
- [ ] データが0件の場合、最大件数を超えた場合の挙動が考慮されている
- [ ] ファイル名の命名規則（タイムスタンプ含む）が定義されている
- [ ] 項目説明（エンドユーザー向けマニュアル）が作成されており、IF仕様書の項目と整合が取れている
- [ ] 不要な項目、および他ドキュメントから変更のない重複記載を避けている
- [ ] AIによる自動生成内容を人間がレビューし、必要に応じて修正・補足されている
- [ ] ユーザー（角田さん）、PMが見て出力イメージやロジックを理解できる内容になっている
