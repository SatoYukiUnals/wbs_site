# 【API設計書】クォーター編集

## 1. 概要

指定クォーターのタイトル・期間を更新する。admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `PUT`
- **Path:** `/api/v1/projects/{project_id}/quarters/{quarter_id}/`
- **認証:** JWT必須（プロジェクト内 admin 以上）

## 3. リクエスト

### パスパラメータ

#### `project_id`（プロジェクトID）

- **型:** `integer` / **必須:** ○

#### `quarter_id`（クォーターID）

- **型:** `integer` / **必須:** ○

### ボディ

#### `title`（クォータータイトル）

- **型:** `string` / **必須:** ○
- 同プロジェクト内で一意

#### `start_date`（開始日）

- **型:** `date` / **必須:** ○
- YYYY-MM-DD

#### `end_date`（終了日）

- **型:** `date` / **必須:** ○
- YYYY-MM-DD。`start_date` 以降であること

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "id": 1,
  "title": "Q1 改訂",
  "start_date": "2026-04-01",
  "end_date": "2026-07-31",
  "progress": 50
}
```

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| バリデーションエラー | 400 | `ERR_VAL_001` | 入力内容に不備があります | |
| タイトル重複 | 400 | `ERR_VAL_007` | このタイトルは既に使用されています | 自レコード除く |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証・権限確認**
   - `ProjectMember` でリクエスト元の `role` が `admin` 以上であることを確認
     - 未認証：401 `ERR_AUTH_002` を返す
     - 権限不足：403 `ERR_PERM_001` を返す
2. **対象クォーター確認**
   - `Quarter` を `quarter_id` AND `project_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
3. **バリデーション**
   - `start_date` ≤ `end_date` チェック
     - エラー時：400 `ERR_VAL_001` を返す
4. **タイトル重複チェック**（`title` 変更時）
   - 同 `project_id` 内で自レコードを除いた `title` の重複確認
     - 重複：400 `ERR_VAL_007` を返す
5. **DB更新**
   - `Quarter` レコードを UPDATE
     - DBエラー：500 `ERR_SYS_500` を返す
6. **レスポンス生成**
   - 更新後のクォーター情報を返却
