# 【API設計書】プロジェクト一覧API

## 1. 概要

ログインユーザーのテナントに紐づくプロジェクト一覧を、進捗率付きで取得する。論理削除済みは除外。

## 2. エンドポイント

- **Method:** `GET`
- **Path:** `/api/v1/projects/`
- **認証:** 必要（Bearer トークン）

## 3. リクエスト

パラメータなし

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "projects": [
    {
      "project_id": "uuid",
      "name": "ECサイトリニューアル",
      "description": "説明テキスト",
      "start_date": "2026-04-01",
      "end_date": "2026-09-30",
      "progress": 40
    }
  ]
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| 認証なし | 401 | 認証情報がありません |

## 5. 内部処理 / ロジック

1. **テナント確認**
   - JWT の `tenant_id` を取得
2. **データ取得**
   - `project` テーブルから `tenant_id` でフィルタ・`deleted_at IS NULL` で取得
   - `project_member` テーブルでログインユーザーが参加しているプロジェクトのみに絞る
3. **レスポンス**
   - プロジェクト一覧を返す
