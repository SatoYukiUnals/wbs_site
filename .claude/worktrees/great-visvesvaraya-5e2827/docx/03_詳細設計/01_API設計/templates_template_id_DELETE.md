# 【API設計書】テンプレート削除

## 1. 概要

指定テンプレートを削除する。作成者本人または admin 以上のみ実行可能。

## 2. エンドポイント

- **Method:** `DELETE`
- **Path:** `/api/v1/templates/{template_id}/`
- **認証:** JWT必須（作成者本人 or admin 以上）

## 3. リクエスト

### パスパラメータ

#### `template_id`（テンプレートID）

- **型:** `integer` / **必須:** ○

## 4. レスポンス

### 成功 (204 No Content)

レスポンスボディなし

### エラー (4xx / 5xx)

| 状況 | ステータス | エラーコード | メッセージ内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 未認証 | 401 | `ERR_AUTH_002` | 認証が必要です | |
| 権限不足 | 403 | `ERR_PERM_001` | この操作を行う権限がありません | |
| 対象未存在 | 404 | `ERR_DAT_404` | 対象のデータが見つかりませんでした | |
| サーバーエラー | 500 | `ERR_SYS_500` | システムエラーが発生しました | |

## 5. 内部処理 / ロジック

1. **JWT検証**
   - `user_id`・`role`・`tenant_id` を取得
     - 無効：401 `ERR_AUTH_002` を返す
2. **対象テンプレート確認**
   - `Template` を `template_id` で SELECT
     - 未存在：404 `ERR_DAT_404` を返す
3. **権限チェック**
   - `created_by = user_id` または `role` が `admin` 以上であることを確認
     - 権限外：403 `ERR_PERM_001` を返す
4. **DB削除**
   - `Template` レコードを DELETE
     - DBエラー：500 `ERR_SYS_500` を返す
5. **レスポンス生成**
   - 204 No Content を返却
