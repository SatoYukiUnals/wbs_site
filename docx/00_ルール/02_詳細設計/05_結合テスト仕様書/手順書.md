# 詳細設計：結合テスト仕様書

## 1. ■概要・目的

複数のモジュール・画面・APIを組み合わせた機能単位で、設計通りに動作することを検証します。本プロジェクトでは、**「テスト項目一覧」** と **「データ一覧」** の2ドキュメントで構成し、テストケースと前提データを分離して管理します。

## 2. ■事前準備

- [ ] 規定のテンプレート：[テスト項目一覧_template.md](./テンプレート/テスト項目一覧_template.md) / [データ一覧_template.md](./テンプレート/データ一覧_template.md)
- [ ] 操作種別ごとの汎用サンプル：[一覧](./一覧/item_list_sample.md) / [変更](./変更/手順書.md) / [詳細](./詳細/item_detail_sample.md) / [登録](./登録/02_手順書.md)
- [ ] 詳細設計（API、画面、機能、バッチ等）の完了
- [ ] テストデータのバリエーション（正常・異常・境界値）の整理

本仕様書は以下のドキュメント構成で作成する：

1. **テスト項目一覧** ([テンプレート](./テンプレート/テスト項目一覧_template.md))：項番順にテストを実施するためのメインドキュメント
2. **データ一覧** ([テンプレート](./テンプレート/データ一覧_template.md))：テスト実施の都度データ生成をしないよう、**事前に作成しておく前提データ** を箇条書きで列挙する（期待値はテスト項目一覧側に記載）
3. **フィールド別ファイル**（任意）：初期表示・バリデーション等で対象フィールドと期待値を全列挙する場合は、テスト項目一覧から参照する別ファイルとして切り出す
4. **（任意）テスト項目一覧（網羅）**：テストケースが多く一覧で俯瞰したい場合、別ファイルで表形式にまとめる

## 3. ■作業手順（Issueのタスクリストとして利用）

- [ ] **データ一覧の作成**：[データ一覧_template.md](./テンプレート/データ一覧_template.md) をコピーして、テスト実施前に **事前に作成しておく前提データ** を箇条書きで列挙します。各ケースには `Case N` 形式のIDを振ります（期待値はテスト項目一覧側に記載）。
- [ ] **テスト項目一覧の作成**：[テスト項目一覧_template.md](./テンプレート/テスト項目一覧_template.md) をコピーして、テストケースごとに「前提条件」「テスト内容」「期待結果」を記載します。前提条件はデータ一覧の `Case No.` を参照します。
- [ ] **フィールド別ファイルの作成**（任意）：初期表示・バリデーション等で対象フィールドが多い場合は、フィールド名・期待値を全列挙した別ファイルを作成し、テスト項目一覧から参照します。
- [ ] **境界値・異常系の網羅**：最大・最小値、NULL値、不正データ、桁数制限の境界等を網羅します。
- [ ] **（任意）網羅一覧の作成**：テストケースが多く、全体を俯瞰したい場合は別ファイルで表形式の一覧を作成します。

## 4. ■記載ルール

- テスト内容・期待結果に**画面仕様書・設計書への参照は記載しない**。期待結果は具体的な値・状態・HTTPステータスで記述する。
  - NG：「仕様書の定義と一致していること」
  - OK：「`name` カラムに入力した値が保存されていること」「200 OK が返却されること」
- テスト共通のドキュメント（データ一覧の `Case No.` 等）への参照はこの限りではない。
- **期待結果はコードでテストが書けるレベルの具体性で記述する**。「正しく表示されること」「正しい値が入っていること」のような曖昧な表現は使わない。フィールド名・値・型・条件を明示する。
  - NG：「初期値が正しく表示されていること」
  - OK：「`order_no` フィールドが空文字で表示されること」「`status` ドロップダウンの選択肢に `ordered`・`shipping`・`completed` が表示されること」
- **テスト中にDBを直接参照・確認しない**。登録・更新の結果確認は詳細画面・一覧画面等のGUIで行う。
  - NG：「データベースに1件登録されていること」「DBの値が更新されていること」
  - OK：「詳細画面に入力した値が表示されていること」「一覧画面に登録したデータが表示されていること」

## 5. ■セルフチェック項目（作業完了後の最終確認）

- [ ] 「テスト項目一覧」と「データ一覧」の2ドキュメントが作成されている
- [ ] 正常系・異常系・境界値が網羅されている
- [ ] 各テストケースの「期待値」が具体的（画面上の表示値・エラーメッセージ・HTTPステータス等）である
- [ ] 使用するテストデータ（ID、件数、値）がデータ一覧に定義されており、テスト実行前に作成可能な形で記載されている
- [ ] 全ての分岐条件（C1/C2）および境界値分析（最大・最小・無効値等）を網羅している
- [ ] テスト項目一覧の「前提条件」とデータ一覧の `Case No.` が整合している
- [ ] 期待値はテスト項目一覧の「期待結果」に記載されており、データ一覧には含めていない
- [ ] 期待結果がチェックボックス形式で記載され、実施時にそのままチェックできる
- [ ] テンプレートの `> [記]` で始まる行をすべて削除している
- [ ] 不要な項目、および他ドキュメントから変更のない重複記載を避けている
- [ ] AIによる自動生成内容を人間がレビューし、必要に応じて修正・補足されている
- [ ] ユーザー（角田さん）、PMが見てテスト内容を理解できる内容になっている
