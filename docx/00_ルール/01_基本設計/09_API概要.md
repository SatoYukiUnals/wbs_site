# 機能仕様書：API概要

## 1. ■概要・目的

対象機能で利用するAPIの全体像と役割を定義します。機能仕様書は**概念レベル**で記述するドキュメントのため、APIパス・HTTPメソッドなどの実装詳細は記載せず、**API名と「何をするAPIか」を日本語で記述**することに留めます。パス・メソッド・リクエスト/レスポンスの項目定義は詳細設計（API設計書）で定義します。

## 3. ■作業手順（Issueのタスクリストとして利用）

- [ ] **APIの列挙**：シーケンス図に登場するAPIをすべてリストアップします。
- [ ] **API名の記述**：各APIに「〇〇一覧取得」「〇〇登録」等、日本語で**何をするAPIか**が分かる名称を付けます。
- [ ] **役割の記述**：各APIが「何のテーブル・対象に対して」「何をするか」を日本語で簡潔に記述します。

## 4. ■セルフチェック項目（作業完了後の最終確認）

- [ ] シーケンス図（04）に登場するAPIがすべて記載されている
- [ ] API名が日本語で、「何をするAPIか」が一目で分かる
- [ ] 役割欄が日本語で具体的に記述されている
- [ ] **APIパス（`/api/v1/...` 等）は記載していない**（詳細設計で定義）
- [ ] **HTTPメソッド（GET/POST/PUT/DELETE）は記載していない**（詳細設計で定義）
- [ ] **英語のenum名・クラス名（`ApprovalStatus`・`UNAPPROVED` 等）は記載していない**（日本語で表現する）
