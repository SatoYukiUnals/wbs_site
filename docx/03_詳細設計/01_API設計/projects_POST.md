# 【API設計書】プロジェクト作成API

## 1. 概要

プロジェクトを新規作成する。作成者は自動的に `admin` ロールでメンバーに追加される。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/projects/`
- **認証:** 必要（Bearer トークン）
- **権限:** admin 以上

## 3. リクエスト

### ボディ（JSON）

#### `name`

- **型:** `string` / **必須:** ✅
- 1〜200文字

#### `description`

- **型:** `string` / **必須:** -
- 説明文

#### `start_date`

- **型:** `date` (YYYY-MM-DD) / **必須:** ✅

#### `end_date`

- **型:** `date` (YYYY-MM-DD) / **必須:** ✅
- `start_date` より後であること

```json
{
  "name": "ECサイトリニューアル",
  "description": "フルリニューアルプロジェクト",
  "start_date": "2026-04-01",
  "end_date": "2026-09-30"
}
```

## 4. レスポンス

### 成功 (201 Created)

```json
{
  "project_id": "uuid",
  "name": "ECサイトリニューアル",
  "description": "フルリニューアルプロジェクト",
  "start_date": "2026-04-01",
  "end_date": "2026-09-30",
  "progress": 0
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| `end_date` ≤ `start_date` | 400 | 終了日は開始日より後に設定してください |
| 必須項目未入力 | 400 | 入力内容に不備があります |
| 権限不足 | 403 | この操作を行う権限がありません |

## 5. 内部処理 / ロジック

1. **権限チェック**
   - JWT の `role` が `admin` 以上か確認。それ以外：403 を返す
2. **バリデーション**
   - 終了日 > 開始日チェック。エラー：400 を返す
3. **DB操作（トランザクション）**
   - `project` レコードを `tenant_id` 付きで INSERT
   - `project_member` レコードを作成者・`role=admin` で INSERT
4. **レスポンス**
   - 作成したプロジェクト情報を 201 で返す
