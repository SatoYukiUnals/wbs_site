# 【API設計書】トークンリフレッシュAPI

## 1. 概要

有効なリフレッシュトークンを受け取り、新しいアクセストークンを発行する。

## 2. エンドポイント

- **Method:** `POST`
- **Path:** `/api/v1/auth/token/refresh/`
- **認証:** 不要

## 3. リクエスト

### ボディ（JSON）

#### `refresh`

- **型:** `string` / **必須:** ✅

```json
{
  "refresh": "<JWT refresh token>"
}
```

## 4. レスポンス

### 成功 (200 OK)

```json
{
  "access": "<新しい JWT access token>"
}
```

### エラー (4xx)

| 状況 | ステータス | メッセージ内容 |
|---|---|---|
| トークン無効・期限切れ | 401 | トークンが無効または期限切れです |

## 5. 内部処理 / ロジック

1. **リフレッシュトークン検証**
   - simplejwt でトークンを検証
   - ブラックリスト確認
   - 無効・期限切れ：401 を返す
2. **アクセストークン発行**
   - 新しいアクセストークンを生成して返す
