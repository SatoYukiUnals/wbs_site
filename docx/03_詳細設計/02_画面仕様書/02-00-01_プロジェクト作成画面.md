# 【画面仕様書】プロジェクト作成画面

## 1. 基本情報

| 項目 | 内容 |
| :--- | :--- |
| 画面ID | 02-00-01 |
| 画面名 | プロジェクト作成画面 |
| URL | `/projects/new` |
| 対応デザイン | - |

## 2. 目的

新しいプロジェクトを作成する。admin以上のみ操作可能。

## 3. 処理概要

### 画面で操作できる情報

- プロジェクト名・説明・開始日・終了日の入力
- プロジェクトの新規作成

### 画面の遷移

| 遷移元 | 操作 | 遷移先 |
| :--- | :--- | :--- |
| プロジェクト一覧画面 | 新規作成ボタン | プロジェクト作成画面 |
| プロジェクト作成画面 | 作成成功 | プロジェクト詳細画面 |
| プロジェクト作成画面 | キャンセルボタン | プロジェクト一覧画面 |

## 4. アクセス権限

## 7. バリデーション

| 項目名 | トリガー | チェック内容 | エラーID | エラーメッセージ |
| :--- | :--- | :--- | :--- | :--- |
| プロジェクト名 | submit | 必須・1〜200文字 | ERR_VAL_001 | プロジェクト名を入力してください |
| 開始日 | submit | 必須・日付形式 | ERR_VAL_001 | 開始日を入力してください |
| 終了日 | submit | 開始日以降（入力時） | ERR_VAL_001 | 終了日は開始日以降の日付を入力してください |

## 8. アクション

### 作成ボタン押下時

1. フロント側バリデーション実行
2. `POST /api/v1/projects/` 呼び出し
3. 成功（201）: `/projects/{id}` へ遷移
4. 失敗（400）: エラーメッセージを表示
